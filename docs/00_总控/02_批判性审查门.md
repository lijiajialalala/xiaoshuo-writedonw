# 批判性审查门

## 目的

这个文档用于给小说项目加一层强制反问。它不负责产出新设定，也不负责证明我们已经对了。它只做一件事：阻止我们把未经验证的偏好、直觉和讨论结论当成事实。

所有关键节点都必须先回答一个问题：

```text
为什么这个判断不是我们在自嗨？
```

## 审查原则

1. 任何方向判断必须有平台证据、样本证据或明确的待验证状态。
2. 任何“热门”“适合”“可改编”“新人能写”都不能直接当结论。
3. 如果一个结论没有反例、边界和失败模式，它还不是结论。
4. 如果一个流程不能产出可验证结果，它只是方法论装饰。
5. 如果一个模型不能 clean-room 生成并通过盲审，它不能升级为 runtime skill。
6. 如果一条机制没有通过合规与相似风险边界检查，它不能进入运行包。

## 立项前审查

### 平台选择

必须回答：

```text
为什么目前无脑番茄，而不是只是因为信息最多、讨论最多、心理上最确定？
```

最低证据：

1. 番茄当前机制和收益规则的来源。
2. 新人进入成本和风险。
3. 同赛道样本在番茄的可见度。
4. AI 使用风险和合同强度。
5. 与其他平台相比，番茄的优势是否仍成立。

不能通过的表现：

1. 只说“番茄流量大”。
2. 只说“大家都这么选”。
3. 没有合同、收益、AI 风险的复核。

### 题材选择

必须回答：

```text
为什么都市高武/异常事件适合我们，而不是我们只是觉得它好玩？
```

最低证据：

1. 榜单样本数量。
2. 前 10 章可执行结构。
3. 能否持续生产事件链。
4. 是否适合我们的资料能力和写作节奏。
5. 是否有改编潜力。

不能通过的表现：

1. 只列设定，不列事件生产机制。
2. 只说“这个题材火”。
3. 能力体系很丰富，但不知道每卷写什么。

### 样本选择

必须回答：

```text
为什么这些样本能代表这个 route？
```

最低证据：

1. 样本来源清楚。
2. 采样日期清楚。
3. 标签、榜单、热度或平台入口清楚。
4. 至少能区分头部样本、中腰部样本、疑似噪声样本。
5. 样本不能只选我们喜欢看的书。

不能通过的表现：

1. 只拿一两本熟悉作品推导赛道。
2. 只选成功样本，不看失败或中腰部样本。
3. 没有记录采样时间，后续无法复核。

## 证据抽取审查

每条证据必须回答：

```text
它在第几章、哪个事件、什么位置发生？
它改变了什么？
它服务哪个生成部件？
如果删掉它，后面是否会断？
它是可迁移机制、类型公约、仅观察、禁止迁移，还是需要风险复核？
```

有效证据格式：

```text
第 X 章，因为 A，主角必须做 B；但 C 阻止他；他采取 D；结果 E 改变了处境；这个变化逼出下一章 F。
```

无效证据格式：

```text
节奏很快。
主角很聪明。
反派有压迫感。
设定很适合改编。
```

这些都是评价，不是证据。

## 归纳审查

每条归纳必须回答：

```text
这个规则来自几条证据？
覆盖几本书？
是否只属于某一本书或某个作者？
有没有反例？
如果用它生成新开局，能不能验证？
```

可以进入 runtime skill 的规则：

```text
在异常调查 route 中，前 3 章通常要完成“异常出现 -> 主角被迫介入 -> 主角发现规则漏洞”的小闭环；否则读者只看到设定，没有看到破局能力。
```

不能进入 runtime skill 的规则：

```text
异常调查要有悬疑感。
主角要聪明。
节奏要快。
```

这些太抽象，不能指导生成。

## 生成审查

生成前必须回答：

1. 这一版激活的是哪个 route。
2. 主读者承诺是什么。
3. 前 10 章兑现点在哪里。
4. 第一卷事件链如何闭合。
5. 阻力如何反咬主角。
6. 翻牌点如何改变局势。
7. 哪些素材只是支撑物，不能喧宾夺主。
8. 哪些地方最可能自嗨。

如果回答不出来，不进入正文。

## 盲审审查

盲审不能只问“像不像番茄”。必须追问：

1. 哪一章让你想继续看。
2. 哪一章开始觉得拖。
3. 哪个冲突是硬塞的。
4. 哪个对手没有行动能力。
5. 哪个翻牌没有改变局势。
6. 哪个设定只是作者想展示。
7. 如果这是新人新书，你愿不愿意追到第 10 章。

盲审反馈只能作为校准证据，不能直接变成模型规则。每次修改 runtime skill 前，必须回到样本证据确认。

## 升级门槛

| 阶段 | 可以进入下一步的条件 |
| --- | --- |
| 样本收集 -> 证据抽取 | 样本来源、日期、标签、章节范围清楚 |
| 证据抽取 -> source-open 归纳 | 证据能定位到章节和结构功能，并标注迁移边界 |
| 单书归纳 -> route 归纳 | 至少 3 本同 route 样本支持 |
| route 归纳 -> runtime skill | 有稳定规则、反例边界、失败模式，且不依赖禁止迁移内容形状 |
| runtime skill -> clean-room 生成 | 能写出前 10 章计划和第一卷事件链 |
| clean-room 生成 -> 盲审 | 生成稿不搬运原文表达，不复用高识别桥段、人设、能力、组织和文风 |
| 盲审 -> skill 更新 | 盲审问题能回到样本证据确认 |

## 当前最高风险假设

1. “番茄就是最优平台”仍需要用最新规则、收益、AI 风险持续复核。
2. “都市高武热门”不等于“异常事件高武适合我们写”。
3. “多能力体系更精彩”可能变成设定拼盘。
4. “借鉴文案蒸馏流程”不等于“小说也能同样复刻成功”。
5. “适合短剧/漫画”不能只靠想象，必须回到场面、人物、钩子和视觉化证据。
6. “前 10 章拆得出来”不等于“100 万字能写下去”。
7. “多源融合”不等于天然安全，仍可能变成多源洗稿或高识别内容拼盘。

这些假设在没有证据前，都只能标记为待验证。
