当前阶段不是写正文，也不是完整搭建小说 Agent。当前阶段只有一个目标：

```text
用最小证据闭环，判断“番茄都市高武/异常事件方向”是否值得进入立项。
```

P2-alpha 复盘见：

```text

```

如果这个闭环跑不通，后面做再多 skill、schema、流程都是空转。

如果同时出现多个都重要的模块，比如编辑门、大纲、文风、长期一致性、平台收益或相似风险，先看 [07_模块依赖与推进顺序.md](07_模块依赖与推进顺序.md)。总控路线图只决定 P0-P8 阶段顺序，模块调度板决定哪些现在做、哪些只留接口、哪些触发后再做。

## 当前必须回答的 6 个问题

1. 番茄是否仍然是当前最优先平台。
2. 番茄、七猫、起点的投稿路径和收益机制分别要求我们生产哪种文本。
3. 都市高武/异常事件方向是否真实有样本支撑，而不是我们主观觉得好玩。
4. 这个方向的前 10 章留人机制是什么。
5. 我们能否 clean-room 写出一个不自嗨、有追读的前 3 章/前 10 章计划。
6. 前 10 章是否已经露出编辑视角的大纲能力、平台安全性和长线可持续性。

暂时不回答：

1. 具体主角最终叫什么。
2. 百万字大纲怎么排。
3. 完整世界观怎么设计。
4. 是否马上建立完整作者 skill。
5. 是否马上写正文。

## 单一主流程

后续只按这条流程推进：

```text
P0 假设登记
-> P0.5 合规与相似风险边界
-> P0.6 三平台机制研究
-> P1 L0 样本池
-> P2-alpha 3 本前 10 章校准
-> P2-beta 10-15 本前 10 章正式快拆
-> P2.5 路线候选判断
-> P3 L2 深拆
-> P4 route 归纳
-> P5 内容内核卡 + runtime skill 草案
-> P6 clean-room 开局诊断
-> P7 盲审回修 + house style 校准
-> P8 立项或放弃
```

## 阶段定义

| 阶段 | 要做什么 | 产出 | 通过条件 |
| --- | --- | --- | --- |
| P0 假设登记 | 把平台、题材、改编、AI 风险假设列出来 | 假设清单 | 每个假设有证据需求或待验证状态 |
| P0.5 合规与相似风险边界 | 定义硬风险、软风险、类型公约和禁止迁移内容形状 | 临时红线和迁移边界标签 | 每条后续证据都有迁移边界 |
| P0.6 三平台机制研究 | 对比番茄、七猫、起点的投稿路径、收益来源、推荐机制、更新成本和违规损失 | 三平台够用表 | P2-alpha 能使用平台字段，不再被平台研究阻塞 |
| P1 L0 样本池 | 收 20-30 本番茄同赛道书 | 样本登记表 | 来源、标签、简介、采样日期清楚 |
| P2-alpha 校准 | 先选 3 本做前 10 章快拆，测试字段是否有效 | 3 本证据包、字段回修记录 | 能判断提取器是否能产出 route_candidate；否则回修协议 |
| P2-beta 快拆 | P2-alpha 通过后，对 10-15 本做前 10 章快拆 | 前 10 章证据表 | 能看出承诺、兑现、因果、阻力、编辑审稿信号、长线大纲苗头 |
| P2.5 路线候选判断 | 判断哪些路线值得深拆，哪些样本只做观察 | route_candidate | 至少有证据支持，且通过相似风险和编辑门初筛 |
| P3 L2 深拆 | 对 3-5 本读前 30 章或完整事件卷 | 单书 source-open 备忘 | 能解释事件链和翻牌系统 |
| P4 route 归纳 | 横向归纳 2-4 条 route | route 归纳表 | 每条 route 至少 3 个样本或明确标为假设 |
| P5 内容内核卡 + skill 草案 | 把最强 route 背后的可迁移生成机制写清，并形成 runtime 生成协议 | 内容内核卡、runtime skill draft | 每条规则有证据编号，且不依赖单书高识别内容形状 |
| P6 clean-room 开局诊断 | 写前 3 章和前 10 章计划，用来诊断 route/内核/叙事身体是否能跑 | 新开局测试稿 | 不搬运原文，不追求像某本书，承诺和因果能跑 |
| P7 盲审回修 + house style 校准 | 做追读/自嗨/平台感/相似风险审查，并校准我们自己的叙述声音 | 盲审记录、house style 草案 | 问题能回到证据或 skill 槽位，且生成稿不像任何单一样本贴皮 |
| P8 立项决策 | 决定写、改方向、还是放弃 | 立项结论 | 有明确继续/停止理由 |

注意：这里不是开始写百万字大纲，而是从前 10 章反推样本是否具备第一卷方向、长期压力、资源/关系循环和编辑审稿潜力。

P2-alpha 三本样本必须覆盖：

```text
1 本目标方向源头/开山/强影响样本
1 本官方榜单当前高热样本
1 本官方榜单中与目标高武系统/升级路线最直接相关的样本
```

目的不是选最喜欢的书，也不是选“结构可拆”的低热书，而是测试提取器能不能顶住源头机制、当前平台偏好和高武系统爽文三种压力。

强作者、强 IP、奖项或改编标杆样本可以进入 `top_benchmark_observe`，但默认不占 P2-alpha 名额，避免把顶级作者势能误判成新人可执行机制。

每本只记录：

```text
书名
作者
平台
来源
采样日期
状态
字数
标签
简介承诺
为什么入选
可能偏差
```

## 暂停事项

在 P2/L1 没完成前，暂停：

1. 写正文。
2. 设计完整世界观。
3. 创建作者级 skill。
4. 讨论最终主角名和细节人设。
5. 把多能力体系当默认方案。
6. 直接规划百万字结构。
7. 把任何单一样本的内容形状迁移进我们的项目。
8. 把 clean-room 开局测试当成最终正文。
9. 把盲审通过理解成“可以模仿某本书/某作者”。
10. 在没有 house style 前进入大规模正文生产。

## P2-alpha 停止条件

P2-alpha 三本校准后，必须先判断，不允许自动扩到 P2-beta。

停止或回修条件：

1. 如果 3 本样本都拆不出稳定的前 10 章留人结构，停止扩大样本，先换 route 或回修 P2 字段。
2. 如果只能拆出设定，拆不出主角行动发动机，暂停异常事件方向，不进入 P3。
3. 如果只能拆出单书高识别桥段，不能压缩成机制，不进入 runtime skill。
4. 如果平台机制显示该方向对新人、AI 辅助或更新成本不友好，降低番茄优先级，重新比较七猫/起点。
5. 如果证据普遍缺少 `transfer_boundary` 和 `risk_note`，停止归纳，先修证据质量。
6. 如果长期一致性需求无法被 E11 记录清楚，不进入 clean-room 前 10 章计划。

## 总控判断

目前项目最大风险不是“不够复杂”，而是太早进入复杂。

正确顺序应该是：

```text
先证明这个方向能留人
再证明我们能写
再证明它能长期连载
最后再扩展成完整创作系统
```

补充一条来自文案系统的新原则：

```text
样本是显微镜，不是目标。
clean-room 是诊断，不是生产。
route 归纳之后必须沉淀内容内核，并转入我们自己的 house style。
```

如果顺序反了，项目会变成设定和方法论都很丰富，但没有一本能开写的书。
