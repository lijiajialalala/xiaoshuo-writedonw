# 全流程批判性复盘

## 结论

当前规划有价值，但问题也很明确：我们把短视频文案蒸馏的成功经验迁移过来以后，快速产生了大量“看起来正确”的流程词，包括证据抽取、source-open、route selector、runtime skill、clean-room、盲审回修。

这些词本身没错，但如果没有总控、阶段门槛和实际样本，它们会变成新的自嗨。

真正需要修的是 6 个问题：

1. 缺总控路线图。
2. 方法论文档过多，执行入口不够唯一。
3. 样本采集没有先定义代表性和偏差。
4. 证据抽取和归纳已经写清，但缺“什么时候够了”。
5. 盲审回修还没有变成操作协议。
6. 当前讨论容易过早跳到设定和世界观。

后续复盘补充：小说项目还必须前置合规与相似风险边界。否则证据抽取会混入高识别内容形状，导致后续 clean-room 生成出现换皮风险。

## 问题 1：没有总控路线图

### 为什么危险

没有总控时，每个文档都像是对的，但不知道下一步到底做什么。结果会变成：

```text
今天补 extractor
明天补 route
后天补 skill
但样本还没收
方向还没验证
```

这会复制文案项目的优点，也复制它的成本。小说项目还没证明方向能跑，就先背上了完整系统的复杂度。

### 修正

新增 [00_当前总控路线图.md](00_当前总控路线图.md)，把当前阶段锁定为：

```text
用最小证据闭环，判断番茄都市高武/异常事件方向是否值得进入立项。
```

并定义 P0-P8 阶段：

```text
P0 假设登记
P0.5 合规与相似风险边界
P1 L0 样本池
P2 L1 前 10 章快拆
P2.5 路线候选判断
P3 L2 深拆
P4 route 归纳
P5 runtime skill 草案
P6 clean-room 开局测试
P7 盲审回修
P8 立项或放弃
```

## 问题 2：执行入口不够唯一

### 为什么危险

旧结构里的 `01-05` 都有道理，但使用者很容易不知道先看哪个。尤其小说专用生成部件流程很长，容易让人误以为下一步要完善所有部件。

这会导致“为了流程完整而工作”，不是为了验证方向而工作。

### 修正

入口顺序改成：

1. `00_当前总控路线图.md`
2. `01_全流程批判性复盘.md`
3. `04_合规与相似风险边界.md`
4. `../10_番茄专项/02_证据提取与归纳规格.md`
5. `02_批判性审查门.md`
6. `../20_小说蒸馏方法/` 背景方法文档

`01-04` 中只有总控、合规边界和证据规格是当前执行入口；其他文档作为背景、方法解释和审查约束。

## 问题 3：样本采集还没有代表性意识

### 为什么危险

如果样本只来自我们熟悉的书、喜欢的书、讨论里提到的书，就会发生样本偏差。最后归纳出的不是番茄赛道机制，而是我们的阅读偏好。

典型风险：

```text
只拿《神秘复苏》《老千》这类强记忆作品做参照
-> 误以为平台读者都吃强智斗和强规则
-> 忽略番茄真实留存可能更依赖快爽和强标签
```

### 修正

L0 样本池必须记录：

```text
来源
采样日期
榜单/搜索/推荐入口
标签
状态
字数
简介承诺
为什么入选
可能偏差
```

并要求区分：

```text
头部样本
中腰部样本
噪声样本
我们主观喜欢但不一定代表平台的样本
```

## 问题 4：证据抽取清楚了，但晋级条件还不够硬

### 为什么危险

我们现在知道要抽：

```text
promise
chapter_motion
plot_causality
opposition
reveal_flip
support_function
renderer
adaptation
```

但如果没有“什么时候够了”，执行时会无限抽，或者抽一点就急着归纳。

### 修正

采用 L0-L3 分级：

```text
L0：20-30 本，只做样本登记。
L1：10-15 本，前 10 章快拆。
L2：3-5 本，前 30 章或完整事件卷深拆。
L3：1-2 本，核心对标，进入 skill 草案。
```

核心判断：

```text
L1 都不能证明方向有前 10 章留人结构，就不进入 L2。
```

## 问题 5：盲审回修还不够操作化

### 为什么危险

如果盲审只问“像不像番茄”“想不想看”，反馈会很虚。虚反馈不能回修 skill。

无效反馈：

```text
感觉还行。
不够爽。
有点慢。
不像番茄。
```

有效反馈应该能定位：

```text
第 2 章没有新压力。
第 3 章主角行动不是被逼出来的。
第 5 章翻牌只解释过去，没有改变局势。
对手只嘲讽，没有主动干预。
```

### 修正

盲审必须按断点提问：

1. 哪一章开始想继续看。
2. 哪一章开始拖。
3. 哪个情节是硬接。
4. 哪个阻力没有行动能力。
5. 哪个翻牌没有改变局势。
6. 哪个设定只是展示。
7. 如果是新人新书，愿不愿意追到第 10 章。

盲审反馈不能直接改 skill，必须回到样本证据确认。

## 问题 6：设定讨论过早

### 为什么危险

都市高武、异常事件、修仙、异能、巫蛊、苗疆、西藏、新疆、张家界这些都很有吸引力，但如果前 10 章留人机制还没验证，设定越多越容易遮蔽问题。

尤其“多能力体系”是高风险假设：

```text
好处：新鲜感强，事件丰富。
风险：规则不统一，变成素材拼盘，读者不知道主角到底在玩什么游戏。
```

### 修正

在 P2 前，所有设定都只能作为候选素材，不能作为立项核心。

立项核心必须先回答：

```text
读者为什么点进来？
前 3 章看到什么？
前 10 章兑现什么？
主角凭什么赢？
对手如何反咬？
第一卷如何闭合？
```

## 当前规划中做得对的地方

1. 已经明确不直接套文案模型，而是改成小说多颗粒度 source-open。
2. 已经区分单书层、route 层、作者层，避免一开始就仿作者。
3. 已经把“读者承诺、叙事因果、阻力、翻牌”列为小说专用生成部件。
4. 已经意识到改编潜力要前置考虑，但没有让它压过小说追读。
5. 已经明确不提交大体量原文，保持 clean-room 意识。

## 当前必须停止的做法

1. 停止继续扩展方法论文档，除非它直接服务 P0.5 合规边界或 P2 提取。
2. 停止讨论完整世界观，直到 L1 前 10 章快拆完成。
3. 停止把“异常事件 + 多能力体系”当默认答案。
4. 停止把熟悉作品当平台代表样本。
5. 停止创建 runtime skill，直到 route 横向归纳通过。
6. 停止把单一样本的内容形状迁移进我们的项目。

## 当前必须开始的做法

1. 按 P0.5 合规与相似风险边界给后续证据加迁移标签。
2. 从 L0 样本池筛 10-15 本做前 10 章快拆。
3. 记录每条证据为什么可迁移、只能观察或禁止迁移。
4. 做 P2.5 路线候选判断。
5. 用证据决定是否进入异常事件 route 深拆。

## 修正后的下一步

马上执行：

```text
P0.5 合规与相似风险边界
P2 L1 前 10 章结构证据
```

不要再继续补与 P0.5/P2 无关的流程。

如果下一步还是继续写泛方法论，说明项目已经开始逃避真实验证。
