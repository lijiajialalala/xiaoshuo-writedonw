# 编辑审稿门与大纲准备

## 0. 结论

需要新增一个 `novel-editor-gate` skill。

它不是最后才出现的“改稿编辑”，而是从 P0 到 P7 贯穿的“编辑门禁”。

原因：

```text
编辑判断会影响平台选择、题材卖点、前3章、前10章、大纲、签约材料、AI风险和相似风险。
如果只在最后出现，很多错误已经写进主角、设定、事件链和文风里，回修成本过高。
```

## 1. 为什么现在需要编辑 skill

当前系统已经有：

- 平台机制研究。
- 合规与相似风险边界。
- P2 前 10 章证据提取协议。
- 内容内核发现流程。
- 跨题材基础叙事逻辑库。

但这些还缺一个贯穿判断者：

```text
这个东西像不像能过编辑/平台/读者第一关？
这个题材、简介、前3章和大纲能不能形成可投稿材料？
这个文本有没有 AI 低质、洗稿、相似、合同或平台风险？
```

所以 `novel-editor-gate` 的定位不是写作，而是审查和放行。

## 2. 公开资料给出的编辑视角

当前已查到的公开信号：

| 来源类型 | 关键信号 | 对项目的意义 |
| --- | --- | --- |
| 番茄公开/媒体报道 | 2025-2026 年持续治理大规模 AI 低质内容、批量水文、同质化严重作品；签约流程在 2万/5万/8万字节点形成评估机会 | 番茄不能只追求“快写”，必须前置 AI低质风险、开篇质量和前 2万字签约可读性 |
| 七猫/奇妙官方福利与投稿页 | 常见投稿包是 2万字正文 + 大纲 + 投稿申请表；审稿流程包含编辑初审、复审、终审；严禁工作室投稿和抄袭 | 七猫更需要编辑材料包，大纲不是可选项 |
| 七猫投诉/登载规则 | 抄袭投诉要求具体到章节和相似情节/语句；高级抄袭包括改头换面窃取受保护独创内容；类型公约、公有素材等可作为例外 | 相似风险检查要覆盖情节顺序、桥段、表达和是否属于类型公约 |
| 起点/作者社区经验 | 内投/直发都看书名、简介、开头正文、题材市场潜力、行文流畅、主线和更新稳定性；大纲常用于投稿和后续评估 | 起点式付费生态更看长期主线可信度和作者稳定性 |
| 七猫写作提升专题 | 开篇看期待感、剧情流畅、逻辑缜密、爆点；新人不要盲目模仿大神开篇 | 编辑门要检查“开篇能不能留人”，不是只看文笔 |

这些信息有官方、媒体、作者经验混合来源，后续要继续在 `docs/30_平台研究/05_来源索引与证据等级.md` 中细化可信度。

## 3. 大纲现在有没有学习

有，但不够 active。

已有位置：

- `docs/20_小说蒸馏方法/03_小说提取归纳总模型.md` 已把大纲和结构层列为运行包核心模块。
- `docs/90_前期规划归档/05_协议模板/story_bible模板.yaml` 已有 Story Bible 模板。
- `docs/90_前期规划归档/05_协议模板/chapter_card模板.yaml` 已有章节策划卡模板。
- `docs/90_前期规划归档/03_创作流程/人类主创_AI执行流程.md` 已有“Story Bible -> context_pack -> chapter_card -> 草稿 -> 审稿”的旧流程。

但问题是：

```text
这些内容目前在总模型或归档里，没有进入当前 active schema。
也就是说，系统知道最终需要大纲，但还没开始真正学习“什么样的大纲能过编辑、能支撑长篇、能进入生产”。
```

## 4. 为什么之前没发现这个缺口

根因不是“大纲不重要”，而是当前阶段目标把注意力压在了 P0-P2：

```text
平台机制
-> 样本池
-> 前10章证据提取
-> route_candidate
```

这导致系统优先问：

- 该拆哪些样本？
- 前 10 章提取哪些证据？
- route 怎么归纳？
- 合规/相似风险怎么前置？

但还没有把“编辑会怎么审一个投稿包”和“大纲怎么作为签约/长线生产材料”单独建模。

所以缺口来自阶段焦点：

```text
我们把大纲当成 P5/P7 以后的产物，
没有把它当成 P2 之后就要开始学习和验证的对象。
```

这个判断需要修正。

## 5. 大纲应该什么时候准备

不是现在就写完整百万字大纲。

但现在必须准备“大纲学习接口”。

阶段安排：

| 阶段 | 大纲相关动作 | 目的 |
| --- | --- | --- |
| P2 前10章快拆 | 记录样本的前10章小闭环、主线承诺、长期压力和未解问题 | 判断这本书是否有大纲感，而不是只靠开头噱头 |
| P2.5 route 判断 | 判断这条 route 是否能支撑第一卷和 30万字以上 | 避免只选会开头、不能长跑的打法 |
| P3 第一卷/事件链深拆 | 学样本如何从前10章延展成第一卷 | 开始学习卷纲、事件链和翻牌分布 |
| P4 内容内核卡 | 把可迁移的大纲机制压缩成内核 | 避免照搬单书卷结构 |
| P5 项目 Story Bible | 准备我们自己的主角、世界、能力、关系、第一卷方向 | 形成编辑可审的项目材料 |
| P6 clean-room 前3章/前10章 | 用前10章计划反测大纲是否可执行 | 防止大纲漂亮但开篇跑不动 |
| P7 盲审/编辑门 | 评估能否投稿、直发或继续重构 | 进入平台前最后回修 |

## 6. 编辑 skill 是否从头贯穿

结论：必须贯穿，但每个阶段职责不同。

| 阶段 | 编辑门职责 |
| --- | --- |
| P0/P0.6 | 查平台规则、收益、AI标识、抄袭处罚、合同和首发独家风险 |
| P1 样本池 | 判断样本是否只符合平台热度，还是有学习价值 |
| P2 快拆 | 标记前3章/前10章是否具备编辑眼里的签约潜力 |
| P2.5 route 判断 | 判断 route 是题材标签、短期噱头，还是可连载打法 |
| P3 深拆 | 看事件链、卷纲和中期留存是否能支撑长篇 |
| P4 内容内核 | 拦截高识别内容形状，防止多源洗稿 |
| P5 Story Bible/大纲 | 判断是否形成可投稿、可执行、可长期连载的大纲材料 |
| P6 clean-room | 判断前3章/前10章计划是否能留人且不贴皮 |
| P7 回修 | 输出 proceed / revise / stop / collect_more_evidence |

它不是“写作 skill”，而是“放行 skill”。

## 7. 编辑门最小检查表

每次调用 `novel-editor-gate`，至少检查：

```text
平台适配：目标平台是什么？直发、内投、签约材料要求是什么？
题材卖点：书名/简介/题材是否能让编辑判断卖点？
开篇质量：前3章是否有期待感、冲突、主角发动机、即时兑现？
前10章：是否完成小闭环，是否能继续追读？
大纲能力：是否能支撑第一卷和30万字以上？
AI风险：是否像批量生成、低质水文、逻辑拼凑、人物不一致？
相似风险：是否像某本书的角色、桥段、组织、能力、事件链或文风？
合同/平台风险：是否涉及首发独家、一稿多投、抄袭、违规内容、AI标识问题？
```

## 8. 需要马上补的东西

1. 新建 `.agents/skills/novel-editor-gate/SKILL.md`。
2. 更新 P2 的 `chapter_evidence.yaml`，加入编辑门和大纲学习字段。
3. 后续把归档里的 `story_bible模板.yaml`、`chapter_card模板.yaml` 升级到 active `schemas/`，但不要现在直接写完整大纲。
4. 下一轮 P2 三本校准样本必须让 `novel-editor-gate` 参与，测试它能不能发现“开篇能跑但长线不能跑”的样本。

## 9. 当前判断

大纲不是现在开始写，而是现在开始学习和建接口。

编辑 skill 不是最后出现，而是贯穿全流程。

下一步不是“马上做百万字大纲”，而是：

```text
把编辑门和大纲学习字段接进 P2，
用 3 本样本校准它是否真的能判断平台、质量、AI风险、相似风险和长线可持续性。
```
