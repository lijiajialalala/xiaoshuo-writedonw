# 人类主创 + AI执行写作流程

更新时间：2026-05-04

## 角色定义

当前更合理的模式不是“人给AI一句意见，AI负责写完”，而是：

`AI做数据策划、章节编剧、草稿执行和风控检查；人类负责方向、讨论、取舍、验收和最终创作判断。`

AI应该每一章先告诉人类它想干什么、为什么这么安排、参考的是哪类爆款结构，而不是直接写正文。

## 标准流程

1. 题材和平台选择

AI分析平台题材、读者预期、爆款结构和风险点。人类决定投番茄还是七猫，决定是否立项。

2. 项目设定

人类确认书名方向、主角、核心卖点、世界观、主线矛盾。AI可以给备选方案，但不能直接替人决定最终设定。

3. 长篇故事圣经

建立 story bible，记录人物、关系、战力、伏笔、重要物品、世界规则、阶段目标。

4. 长篇记忆包

每章写前生成 `context_pack`，只加载本章真正需要的记忆：最近章节摘要、出场人物状态、未解决伏笔、当前冲突、风格约束和不能写错的事实。

5. 每章策划卡

AI每章先输出策划卡，说明章节目标、读者情绪、节奏安排、爽点位置、悬念、不能做什么、需要人类确认的问题。

6. 人类讨论与确认

人类和AI讨论章节策划卡，否决不合理的安排，确认本章方向。

7. AI生成正文草稿

AI根据确认后的策划卡写正文，不直接照搬爆款桥段。

8. 风控和质量检查

检查AI味、重复桥段、人物一致性、节奏拖沓、抄袭/洗稿风险、平台敏感风险。

9. 人类验收

人类判断是否达到章节目的，提出修改方向。具体改写可以继续交给AI执行。

10. 写后记忆更新

更新章节摘要、人物关系、伏笔状态、已使用爽点、重复风险和人类决策记录。

11. 发布与复盘

记录章节数据，观察追读、评论、收藏、推荐变化，反推下一阶段节奏。

## 章节策划卡模板

```text
第X章策划卡

本章位置：
当前处于第几卷/第几阶段/第几个冲突单元。

本章目标：
这一章必须推动什么变化。

读者情绪：
开头、中段、结尾分别要让读者产生什么情绪。

节奏安排：
1. 开头如何承接上一章。
2. 中段如何升级冲突。
3. 爽点或反转放在哪里。
4. 结尾如何制造下一章阅读动机。

人物推进：
本章哪些人物关系发生变化。

伏笔与回收：
本章埋什么，回收什么。

不能做：
哪些底牌不能暴露，哪些人物不能崩，哪些桥段不能重复。

爆款结构依据：
这一章借鉴的是哪类结构规律，而不是哪本具体作品。

需要人类确认：
列出3到5个必须由人类决定的问题。
```

## 人类反馈标准

弱反馈：

- 不够爽，重写。
- AI味太重。
- 女主不讨喜。

强反馈：

- 本章不能让男主彻底赢，只能小胜。
- 反派可以丢面子，但不能下线。
- 女主不能救场，只能观察到一个关键细节。
- 结尾要让读者知道更大的危机来了，但主角暂时不知道。
- 这一章对话太多，改成动作和误会推进。

强反馈能体现人类主创判断，弱反馈更像简单外包。
