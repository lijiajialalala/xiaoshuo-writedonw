# 文笔风格控制协议

更新时间：2026-05-04

## 目的

结构 skill 解决“故事怎么推进”，文笔风格协议解决“文字怎么写得像人、像同一个人、不像默认AI腔”。

文风不能靠模仿某个作者解决。我们的文风来源应该是：

- 人类主创的审美判断。
- `style_bible.yaml` 中明确的风格参数。
- 通用写作技巧。
- anti-AI 表达库。
- 长期修改后沉淀出的稳定样本。

## 基本原则

- 少写抽象情绪，多写动作、停顿、反应和选择。
- 少用总结句，多用事件推进。
- 少解释设定，多让设定在冲突中出现。
- 少堆形容词，多给具体物件和具体动作。
- 对话要有目的、压迫、遮掩或试探，不能只负责说明信息。
- 每章语言风格要稳定，不要突然变成另一种口吻。

## AI腔高风险信号

以下表达要默认警惕：

- 情绪直接总结：愤怒、震惊、复杂、坚定、无奈。
- 空气变化套话：空气凝固、气氛骤然紧张、全场死寂。
- 命运套话：命运齿轮、时代洪流、无人知道。
- 眼神套话：眼神中闪过、眸中掠过、眼底浮现。
- 机械转折：然而、就在这时、下一秒、与此同时连续使用。
- 解释型对话：角色把背景设定完整说给对方听。
- 重复动作：冷笑、攥拳、咬牙、挑眉、眯眼连续滥用。

## 文笔 Skill 标准格式

```yaml
skill_id: prose_001_action_over_emotion
name: 用动作替代情绪直说
skill_type: prose
status: candidate

problem:
  - AI容易直接写愤怒、震惊、屈辱
  - 直接写情绪会降低画面感和人物可信度

use_when:
  - 主角被羞辱
  - 主角压怒
  - 反派试探
  - 女主观察到细节变化

bad_example:
  - 他感到无比愤怒。
  - 她心中涌起一股复杂的情绪。

good_pattern:
  - 他的手指在袖子里慢慢攥紧。
  - 她没有接话，只把杯口转向了自己。

avoid:
  - 连续重复攥拳、咬牙、冷笑
  - 每次情绪都用肢体动作替换，导致动作堆砌
  - 动作和人物性格不匹配

evaluation_checks:
  - 删除情绪词后，读者仍能判断人物情绪
  - 动作服务当前冲突，而不是装饰
  - 同一章内动作表达不重复
```

## 文风审稿流程

1. 句子层检查

检查抽象情绪、套话、重复动作、机械转折、空泛形容词。

2. 段落层检查

检查是否有整段总结、设定解释、AI式鸡汤或作者站出来讲道理。

3. 对话层检查

检查角色说话是否符合身份、关系、目标和当前压力。

4. 场景层检查

检查描写是否服务冲突，是否有过度环境描写和无效氛围描写。

5. 章节层检查

检查本章文风是否与前文一致，是否突然变油、变装、变解释。

## 人类反馈沉淀规则

人类每次指出问题，都应沉淀成 style 规则：

- “这句太AI” -> 增加 forbidden_ai_phrases。
- “这里太油” -> 增加 forbidden_tone。
- “别直接说情绪” -> 增加 prose skill。
- “对话太解释” -> 增加 dialogue anti-skill。
- “这一段像我想要的感觉” -> 加入 preferred_examples。

文风不是一次设定完成，而是靠反复修改形成稳定口味。
