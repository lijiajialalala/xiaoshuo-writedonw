# AI使用红线与处罚风险

更新时间：2026-05-04

## 核心判断

“AI使用”和“抄袭”不是同一个问题。

可以不抄袭但使用AI；但如果AI使用方式导致作品不像本人自主独立创作、像批量低质内容、像洗稿、像旧稿套壳，平台仍可能处理。

## 官方规则层面

番茄小说公开规则重点：

- 平台公开提供AI辅助工具，包括灵感、大纲、起名、扩写、改写、续写、查询等。
- 平台对AI生成低质内容有明确风险提示，可能拒签、下架、多次低质后不再给签约机会。
- 内容安全违规可触发停止推荐分发、取消福利资格、冻结未发稿酬等措施。
- 抄袭、侵权、旧稿挪用、自我抄袭可触发追回违规稿费，严重时退回全部稿酬、封号。

七猫中文网公开规则重点：

- 《作品登载规则和服务约定》要求作品为作者本人自主、独立创作，不存在抄袭、剽窃、权利瑕疵。
- 平台可独立判断并采取拒绝发布、删除内容、暂停更新、冻结权益、永久关号、追责等措施。
- 违约或侵权导致平台损失时，平台可停止支付稿酬、要求退回已收稿酬、直接扣除稿酬并主张违约责任。
- 2025作家签约政策明确严禁工作室投稿，发现后可解除协议、追回稿费、赔偿并追责。
- 七猫也有AI辅助工具，所以不能简单理解为“七猫反AI”，更准确是“七猫对独立创作和合同责任更硬”。

## AI与抄袭的区别

低风险AI使用：

- 查资料。
- 题材分析。
- 起名备选。
- 章节策划。
- 爽点密度分析。
- 生成多个剧情方案供人类选择。
- 草稿生成后由人类讨论、验收、改方向。

高风险AI使用：

- AI直接生成全文，人类只上传。
- AI批量生成多个账号、多本书。
- 拿已有小说喂给AI改写。
- 复用爆款书的主线、人设、关键桥段。
- 投稿表、大纲、卖点直接使用AI生成文本且自己说不清逻辑。
- 同一本书多平台投稿或换名投稿。

绝对不要碰：

- 抄袭、洗稿、融梗到高度相似。
- 旧稿挪用、自我抄袭。
- 同一本书签约后投另一平台。
- 工作室化伪装单作者。
- 多账号批量铺量。

## 风险矩阵

| 行为 | 番茄风险 | 七猫风险 | 判断 |
|---|---:|---:|---|
| AI查资料、做策划 | 低 | 低 | 可做 |
| AI写草稿，人类讨论后定稿 | 中低 | 中 | 可做，但要留证据链 |
| AI直接写正文，人类少量确认 | 中高 | 高 | 不建议 |
| AI批量生成多本书 | 高 | 极高 | 不建议 |
| AI改写已有小说 | 极高 | 极高 | 禁止 |
| 同一本书多平台投稿 | 高 | 高 | 禁止 |
| 不同小说分别投不同平台 | 低 | 低 | 可做 |

## 证据链要求

建议每章保存以下记录：

- 章节策划卡。
- 人类讨论意见。
- AI生成的多个方案。
- 人类选择和否决理由。
- AI初稿。
- 修改稿。
- 最终稿。
- 人设、伏笔、世界观变更记录。
- 相似性和AI味检查结果。

证据链不是为了“证明没用AI”，而是为了证明人类拥有实质创作控制权。
