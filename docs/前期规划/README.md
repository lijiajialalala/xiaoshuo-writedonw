# AI小说创作前期规划

更新时间：2026-05-04

这个目录用于沉淀“AI辅助网文创作”项目前期规划，包括平台策略、规则风险、创作流程和系统设计。当前讨论重点是番茄小说与七猫中文网，不做全网乱投。

## 目录结构

- `00_总览`：当前结论、战略摘要、核心判断。
- `01_平台对比`：番茄小说 vs 七猫中文网专项对比。
- `02_规则与风险`：AI使用、抄袭、工作室、收益追回、限流等风险。
- `03_创作流程`：人类主创、AI数据策划与执行写作流程。
- `04_系统设计`：基于 `douyin-wenan` 的小说系统改造方案。
- `05_协议模板`：小说结构蒸馏、文笔风格控制、长篇记忆管理，以及 Story Bible、Style Bible、章节状态等模板。
- `06_来源索引`：官方规则、媒体报道、作者自述来源。

## 当前总判断

可以做 AI 辅助小说项目，但不建议定义成“纯AI一键出书”。更稳的模式是：

`人类主创 + AI数据策划 + AI执行写作 + 本地主账本 + 本地知识库检索 + 长篇记忆包 + 三层审稿 + 人类验收。`

平台选择上，建议先深耕 1 到 2 个平台：

- 番茄小说：适合测试自然流量、算法反馈和大众题材。
- 七猫中文网：适合测试编辑审稿、签约、保底/分成路径，但合同追责更硬。

不同小说可以分别投番茄和七猫；同一本书、换名旧稿、相似大纲、人设复用、同世界观拆分投稿都不建议。

## 当前补充重点

后续系统不能只做平台风控和章节生成，还要补齐两层：

- 小说结构蒸馏协议：规定从公开样本中只能提炼通用结构机制，不能复用具体桥段、人设、设定和文风。
- 文笔风格控制协议：建立 `style_bible.yaml`、文笔 skill 和 anti-AI 表达库，让正文长期保持稳定语感。
- 长篇记忆管理协议：通过 `context_pack`、章节摘要、人物状态、伏笔列表和人类决策记录，避免 AI 写到后面忘记前文。
- 本地知识库协议：用 SQLite/全文检索/后续向量检索辅助找回旧章节、人物事件、伏笔、规则和风险，但不替代本地主账本。
