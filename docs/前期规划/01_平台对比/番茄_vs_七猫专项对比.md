# 番茄小说 vs 七猫中文网专项对比

更新时间：2026-05-04

## 对比结论

番茄更像“流量试验场”，七猫更像“合同签约场”。如果要用AI辅助创作，番茄适合先跑数据，七猫适合拿更成熟、更人工主导的项目去投。

| 维度 | 番茄小说 | 七猫中文网 |
|---|---|---|
| 平台性质 | 免费阅读、算法推荐、自然流量强 | 免费阅读、编辑审稿和签约路径更重 |
| 自然流量 | 强，但不稳定 | 相对弱于番茄，更多依赖签约和推荐安排 |
| AI公开态度 | 平台公开提供AI灵感、大纲、扩写、改写等工具 | 平台也公开提供AI小助理、AI起名、灵感等工具 |
| AI风险 | 低质AI内容可能拒签、下架、不给量、停推 | 未看到“AI一律违规”的公开条款，但“独立创作”和合同追责空间更大 |
| 处罚风险 | 停止推荐、取消福利、冻结未发稿酬；抄袭可追回违规稿费 | 拒绝发布、删除、暂停更新、冻结权益、停发稿酬、退回稿酬、解约、追责 |
| 限流风险 | 作者自述中更常见：验证期失败、首秀差、书城0量、停推 | 公开作者自述较少，可能更多发生在审稿/签约前 |
| 适合项目 | 强爽点、大众题材、能快速更新和测试 | 大纲扎实、开篇强、人工主创痕迹明显 |
| 不适合 | 高度模板化批量AI文、旧稿套壳 | 纯AI代写、工作室化、投稿表和大纲明显AI生成 |

## 番茄重点

番茄的优势是流量和测试效率。风险也集中在流量侧：作品可能不是马上被判违规，而是没有推荐、首秀失败、验证期数据差、后续停推。

公开规则中，番茄对抄袭、侵权、低质内容都有明确处罚工具。2025年规则中还明确提到，AI生成低质内容可能导致拒签、下架、多次发布低质内容后不再给签约机会。

对番茄的策略：

- 可以用AI做题材分析、章节策划和草稿。
- 正文必须避免模板腔、重复爽点和机械转折。
- 每日更新要稳定，不建议异常爆更。
- 核心考核是读者留存、追读和推荐反馈。

## 七猫重点

七猫公开条款中没有看到“使用AI本身一律违规”的明文，并且平台自己提供AI辅助功能。但七猫的合同追责工具更硬，尤其是“非本人自主独立创作”“工作室投稿”“侵权/抄袭”等认定空间。

七猫2025签约政策明确严禁工作室投稿，定义包括“写同一本书的不止一个作者”。发现后可能解除协议、追回稿费、赔偿并追责。

对七猫的策略：

- 投稿表、大纲、卖点、前2万字必须人工主导。
- AI可以参与策划和草稿，但要有清晰的人类讨论记录。
- 不建议用七猫测试纯AI批量产文。
- 如果签约，一定先看合同里是否有作者独家、作品独家、优先权、衍生权等条款。

## 平台分工建议

番茄项目：

- 目标：测试题材、节奏、自然流量和读者反馈。
- 方法：高频迭代、强数据复盘、章节节奏优化。
- 风险控制：防AI味、防重复、防低质、防异常爆更。

七猫项目：

- 目标：测试签约、保底/分成、编辑判断。
- 方法：低频但高质量投稿，强化大纲和开篇。
- 风险控制：防非独立创作认定、防工作室化、防投稿材料AI痕迹过重。
