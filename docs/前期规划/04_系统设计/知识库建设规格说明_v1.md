# 知识库建设规格说明 v1

更新时间：2026-05-04

## 目标

建设一个本地 LLM Wiki + 本地检索库，服务两个方向：

- 视频文案：继承 `douyin-wenan` 的样本提取、结构分析、skill/anti-skill 沉淀。
- 小说创作：支撑平台风控、结构蒸馏、文风控制、长篇记忆、章节策划和审稿。

核心原则：

`共享知识，不共享项目状态。`

知识库沉淀通用知识；每个具体项目保留独立主账本、独立索引和独立输出。

## 页面类型

### source

用于保存来源材料。

适合内容：

- 番茄/七猫官方规则。
- 媒体报道。
- 社区自述。
- 外部项目说明。
- 公开样本观察笔记。

必要字段：

```yaml
---
type: source
source_id: source_platform_fanqie_ai_rules_001
title: ""
source_url: ""
source_kind: official_rule
credibility: high
domain: platform_policy
related_entities: []
related_concepts: []
created_at: ""
updated_at: ""
---
```

### concept

用于沉淀概念。

适合内容：

- AI味。
- 长篇记忆。
- 追读钩子。
- 洗稿风险。
- 结构蒸馏。
- 文风圣经。

必要字段：

```yaml
---
type: concept
concept_id: concept_longform_memory
name: 长篇记忆管理
domain: novel_creation
status: active
related_sources: []
related_skills: []
created_at: ""
updated_at: ""
---
```

### entity

用于沉淀实体。

适合内容：

- 平台：番茄、七猫、抖音。
- 项目：某本小说、某个视频账号。
- 角色：小说人物。
- 工具：douyin-wenan、chinese-novelist-skill、Hermes。

必要字段：

```yaml
---
type: entity
entity_id: entity_platform_fanqie
name: 番茄小说
entity_kind: platform
domain: novel_platform
status: active
related_sources: []
related_concepts: []
created_at: ""
updated_at: ""
---
```

### skill

用于沉淀可复用方法。

分为：

- structure：结构 skill。
- prose：文笔 skill。
- anti_ai：反 AI 味 skill。
- risk：风控 skill。
- video：视频专用 skill。
- novel：小说专用 skill。

必要字段：

```yaml
skill_id: structure_opening_pressure_001
type: skill
skill_kind: structure
status: candidate
name: 开篇压迫式羞辱
domains:
  - novel
use_when: []
core_mechanism: []
avoid: []
originality_checks: []
evidence_refs: []
```

### playbook

用于沉淀工作流。

适合内容：

- 视频文案分析流程。
- 小说章节策划流程。
- 风控审稿流程。
- context_pack 生成流程。

### decision

用于记录关键决策。

适合内容：

- 为什么先做番茄和七猫。
- 为什么不用 Hermes 作为地基。
- 为什么本地文件是主账本。
- 为什么视频和小说共享知识但隔离项目状态。

## SQLite 最小表

第一阶段只做必要表：

```sql
sources(
  source_id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  source_url TEXT,
  source_kind TEXT,
  credibility TEXT,
  domain TEXT,
  path TEXT NOT NULL,
  created_at TEXT,
  updated_at TEXT
);

concepts(
  concept_id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  domain TEXT,
  status TEXT,
  path TEXT NOT NULL,
  summary TEXT
);

entities(
  entity_id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  entity_kind TEXT,
  domain TEXT,
  status TEXT,
  path TEXT NOT NULL,
  summary TEXT
);

skills(
  skill_id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  skill_kind TEXT,
  status TEXT,
  domains TEXT,
  path TEXT NOT NULL,
  summary TEXT
);

documents_fts(
  doc_id,
  doc_type,
  title,
  body
);
```

要求启用 SQLite FTS5，用于全文搜索。

第二阶段再加：

- chapter summaries。
- character events。
- foreshadowing events。
- review findings。
- vector embeddings。

## 摄入流程

1. 收集来源

把原始链接、摘录、来源类型、可信度记录成 source。

2. 提炼概念

从 source 中提取 concept，例如 AI味、工作室风险、长篇记忆。

3. 建立实体

把平台、工具、项目、角色建成 entity。

4. 蒸馏 skill

只提炼通用机制，不复用具体桥段、人设、台词和文风。

5. 记录决策

关键架构选择和策略选择都写入 decision。

6. 建索引

运行脚本把 wiki 文件写入 SQLite 和 FTS。

## 视频文案与小说隔离规则

共享：

- 通用结构 skill。
- 文笔 skill。
- anti-AI 规则。
- 风控规则。
- 平台政策。
- 样本观察方法。

隔离：

- 视频账号状态。
- 视频脚本文案。
- 小说 story bible。
- 小说人物状态。
- 小说伏笔索引。
- 小说章节正文。

检索时必须带 domain：

- `domain=video`：优先查视频项目和视频 skill。
- `domain=novel`：优先查小说项目和小说 skill。
- `domain=shared`：查通用概念和通用规则。

## 验收标准

第一阶段完成标准：

- 目录结构完整。
- README 说明清楚。
- 至少 5 个 source 页面。
- 至少 5 个 concept 页面。
- 至少 3 个 entity 页面。
- 至少 5 个 skill 页面。
- 至少 3 个 decision 页面。
- SQLite 数据库可生成。
- FTS 搜索可查询。
- 视频和小说有独立项目入口。
