# P0.6 三平台够用表

## 0. 用途

这个文件不是完整平台研究报告。

它只回答一个问题：

```text
当前平台机制，是否已经足够影响 P2-alpha 三本样本校准？
```

如果一个信息不能改变 P2 拆书字段、样本选择、风险判断或后续投稿策略，就先不在这里展开。

## 1. 够用标准

P0.6 不是要把番茄、七猫、起点研究透，而是要补到能支持下一步：

```text
P2-alpha：3 本前 10 章校准批次
```

进入 P2-alpha 前，至少要能回答：

1. 这个平台更偏直发验证、编辑投稿包，还是付费长线。
2. 新人最可能拿到的钱来自哪里。
3. 收益触发条件会怎样反推前 3 章、前 10 章、2 万字和 10 万字。
4. 断更、AI 低质、相似/抄袭、合同独家这些风险是否会改变生产策略。
5. P2 拆样本时必须多看哪些字段。

## 3. 对 P2-alpha 的直接影响

P2-alpha 三本样本校准时，必须把平台机制压到这些字段里：

| P2 字段 | 平台机制要求 |
| --- | --- |
| `platform_submission_path` | 判断样本更像番茄直发验证、七猫编辑投稿包、起点付费长线，还是跨平台候选 |
| `platform_monetization_vector` | 判断样本最可能靠广告听读、VIP订阅、全勤/福利、保底、活动、版权还是用户拉新赚钱 |
| `platform_strategy_note` | 标注这条证据服务签约、推荐、读完率、追读、投稿包、付费追订还是合规审查 |
| `editor_review_signal` | 标注编辑会不会继续看，以及退稿/拒签可能在哪里 |
| `outline_signal` | 标注前 10 章是否能反推第一卷方向和大纲能力 |
| `continuity_memory_signal` | 标注后续哪些状态变量必须外部化记录，防止长篇崩坏 |
| `ai_quality_signal` | 标注是否有批量生成、逻辑拼凑、水文、人物工具化、设定漂移 |
| `transfer_boundary` | 标注能迁移机制、类型公约重构、只能观察、禁止迁移、软风险复核或硬风险复核 |

## 4. P0.6 停止条件

出现以下情况，就停止继续扩平台研究，先进入 P2-alpha：

1. 三个平台已经能区分直发、内投、付费长线的生产压力。
2. 每个平台已经能列出主要收益来源和关键触发节点。
3. AI 低质、抄袭/相似、断更、合同/独家风险已经能进入 P2 字段。
4. 对前3章、前10章、2万字的要求已经能转成拆书问题。
5. 剩余问题主要依赖后台、合同、编辑口径或作者截图，短期公开资料无法解决。

不能因为以下原因继续拖延 P2-alpha：

1. 起点当前福利没完全核实。
2. 七猫保底报价没有真实合同样本。
3. 番茄后台阈值没有截图。
4. 所有活动金额还没穷尽。
5. 还想再看更多论坛评论。

这些问题进入 [04_平台待验证清单.md](04_平台待验证清单.md)，不阻塞 P2-alpha。

## 5. 当前判断

基于现有研究，P0.6 已经够支持 P2-alpha。

下一步不是继续写平台长文，而是：

```text
选 3 本样本
-> 跑 P2-alpha 前 10 章校准
-> 用样本反过来检查平台字段是否够用
```
