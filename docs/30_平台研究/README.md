# 平台研究

## 定位

这个目录用于记录小说生产运行包的外部平台约束。

它不解决“小说怎么写得好”，而是解决：

```text
同一套小说能力，在不同平台上会被什么入口、规则、收益、推荐、合同和风控重新塑形？
```

当前只做三个平台：

```text
番茄
七猫
起点/阅文
```

## 当前结论

新人都需要强开局，但不同平台的强开局服务的目标不同。

| 平台 | 强开局首先服务什么 | 当前判断 |
| --- | --- | --- |
| 番茄 | 2万字签约、8万字推荐评估、7天小流量验证、读完率/追读率 | 最适合先做算法反馈和免费阅读模型验证 |
| 七猫 | 2万字正文+大纲+投稿申请表、编辑初审/复审/终审、保底或分成合同 | 更适合带完整卖点包内投，验证编辑商业判断 |
| 起点/阅文 | 签约、入V、订阅追读、月票、作者品牌和长期付费读者 | 更适合长期付费生态，但公开当前规则需要后台复核 |

## 可信度标签

| 标签 | 含义 |
| --- | --- |
| high | 官方页面、当前仍可访问，且规则文字清楚 |
| medium | 官方页面但时间较旧，或部分关键数据需要后台/合同复核 |
| low | 作者经验、论坛反馈、媒体报道，只能作为线索 |
| pending | 尚未找到可靠来源，必须后续验证 |
