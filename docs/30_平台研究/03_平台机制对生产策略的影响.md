# 平台机制对生产策略的影响

## 0. 总判断

三平台都需要强开局，但强开局不是同一个东西。

| 平台 | 强开局服务对象 | 第一生产目标 |
| --- | --- | --- |
| 番茄 | 编辑签约、算法验证、免费读者留存 | 前2万字签约+8万到10万字推荐验证 |
| 七猫 | 编辑内投、合同模式、保底/分成判断 | 前2万字正文+大纲+明确商业卖点 |
| 起点/阅文 | 编辑、收藏追读、VIP订阅读者 | 强开局+长期主线可信度+入V后追订 |

所以我们不能只做“番茄爆点文”，也不能只做“传统长线大纲”。前期生产系统必须能输出不同平台需要的材料包。

起点当前公开规则不够完整，生产上先按“付费生态对照平台”处理：

1. 不把起点旧福利当2026确定收益。
2. 不把作者圈的内投字数、追读阈值和编辑偏好当官方规则。
3. 只把起点样本用于校验：强开局能否转化为收藏、追读、付费预期和长期主线信任。
4. 正式投起点前，必须补作家助手后台、编辑口径和合同审查。

## 1. 生产顺序调整

原路线是：

```text
P0 假设登记
-> P0.5 合规与相似风险边界
-> P1 L0 样本池
-> P2 L1 前10章快拆
```

现在应插入：

```text
P0.6 三平台机制研究
```

调整后：

```text
P0 假设登记
-> P0.5 合规与相似风险边界
-> P0.6 三平台机制研究
-> P1 L0 样本池
-> P2 L1 前10章快拆
```

原因：

如果不先知道签约、推荐、收益、全勤、断更和违规损失，P2拆书会只拆“好不好看”，而不是拆“为什么这个文本能在某个平台赚钱”。

## 2. P2 拆书新增平台维度

后续 P2 每本样本增加五个字段。

```yaml
platform_submission_path:
platform_monetization_vector:
platform_review_pressure:
platform_update_pressure:
platform_violation_pressure:
```

### 2.1 platform_submission_path

问：

```text
这本书更像服务直发验证、编辑内投，还是付费长线？
```

填写例：

```yaml
platform_submission_path: fanqie_direct_validation
```

可选值：

```text
fanqie_direct_validation
qimao_editor_submission
qidian_paid_longtail
cross_platform_candidate
unknown
```

### 2.2 platform_monetization_vector

问：

```text
这本书最可能通过什么赚钱？
```

可选值：

```text
ad_reading_listening
subscription_vip
full_attendance
editor_guarantee
activity_bonus
tips_gifts
ip_short_drama
ip_comic_audio
platform_new_user_acquisition
```

注意：这不是说它一定能赚到，而是记录样本的收益适配方向。

### 2.3 platform_review_pressure

问：

```text
平台最可能在哪个环节筛掉这本书？
```

可选值：

```text
signing_review
recommendation_evaluation
editor_submission_review
vip_conversion
retention_data
compliance_review
```

### 2.4 platform_update_pressure

问：

```text
这条路线对更新有什么要求？
```

需要记录：

1. 是否依赖日更4000字以上。
2. 是否容易在8万到10万字前断事件链。
3. 是否容易灌水。
4. 是否需要大量资料。
5. 是否适合AI辅助提效。

### 2.5 platform_violation_pressure

问：

```text
这条路线最容易踩什么平台风险？
```

常见风险：

1. 抄袭/融梗。
2. 旧稿挪用。
3. 恶意水文。
4. 重复章节。
5. 异常发文。
6. 工作室/多人共写风险。
7. 内容安全风险。
8. 版权瑕疵。

## 3. 对样本选择的影响

P1样本池不能只收番茄头部。

更合理的样本结构：

| 样本类型 | 用途 |
| --- | --- |
| 番茄同赛道强样本 | 学算法推荐和免费阅读留人 |
| 番茄中腰部样本 | 看普通作者可执行写法，不只学头部幻觉 |
| 七猫同/邻赛道样本 | 看编辑内投、保底、留存型写法 |
| 起点同/邻赛道样本 | 看付费追订和长期设定可信度 |
| 失败或低分样本 | 反向识别自嗨、拖沓、开局失效、AI感 |

当前可以先保留番茄为主样本池，但 P2.5 之前至少要用七猫和起点各找少量反例/对照样本，防止把番茄平台偏好误认为小说能力本身。

## 4. 对“读者承诺模型”的修正

读者承诺不是一句话，也不是只能选一个。

一本长篇可以同时承诺：

```text
爽点
悬疑
翻牌
关系拉扯
成长
压迫感
智斗
世界观探索
```

但每个阶段只能有主承诺。

拆解时要分颗粒度：

| 颗粒度 | 要问什么 |
| --- | --- |
| 单书层 | 这本书总体靠什么留人 |
| 第一卷层 | 第一卷承诺什么小闭环 |
| 前10章层 | 前10章必须兑现什么 |
| 前3章层 | 点击后立刻给什么钩子 |
| 单章层 | 这一章末尾为什么点下一章 |
| 场景层 | 这一场景是加压、兑现、误导、翻牌还是补偿 |

所以 P2 不应该问“这本书到底是爽文还是悬疑”，而应该问：

```text
主承诺是什么？
副承诺是什么？
哪个阶段哪个承诺上前台？
承诺之间如何切换而不散？
```

## 5. 对 AI 辅助创作的约束

平台规则带来的结论很直接：

AI不能作为“高频灌稿机器”。

它只能作为：

1. 样本证据抽取助手。
2. 大纲推演助手。
3. 多版本桥段压力测试助手。
4. 风险检查助手。
5. 文风稳定和反AI感校对助手。
6. 更新计划和素材管理助手。

不能做：

1. 短期高频创建大量作品。
2. 高频发布大量章节测试平台。
3. 用同质化模板批量改写。
4. 把多个样本的内容形状拼接。
5. 生成明显灌水或重复章节。
6. 用AI代替作者对故事承担责任。

平台层硬理由：

1. 番茄2026福利明确提到异常发文行为会终止活动参与资格。
2. 番茄签约流程优化说明明确治理大篇幅使用AI生成的低质内容，并可能拒绝签约、下架内容，多次发布低质内容的作者不再给予签约机会。
3. 番茄内容发布规范把批量重复/高度相似、无意义低质内容、内容拼凑、复制百科新闻教材、逻辑不连贯、流水账、空洞升华、人物前后不一致等列为可处置风险。
4. 番茄抄袭处罚明确包含同义替换、故事背景/人物设定/剧情走向相似等情况。
5. 番茄低俗等违规处罚说明提示，为了维持全勤而上传违规内容、复制粘贴他人或网络平台设定/描写，会影响推荐、福利、作品和账号。
6. 七猫严禁工作室投稿，同一本书不止一个作者会触发风险。
7. 七猫一般收独家首发，外站发过的作品需要删除下架后再投稿，否则可能无法签约。
8. 起点/阅文付费生态和作者品牌对低质AI文的软风险更高。

法律和合规层理由：

1. 《人工智能生成合成内容标识办法》自2025年9月1日起施行，生成合成内容包括文本、图片、音频、视频、虚拟场景等。
2. 该办法要求相关网络信息服务提供者对生成合成内容进行显式/隐式标识，并要求用户发布生成合成内容时主动声明并使用服务提供者提供的标识功能。
3. 小说平台如何在作者后台和作品发布环节落地标识要求，仍需以具体平台协议和后台功能为准。

因此，项目内部要把 AI 辅助拆成三道门：

| 门禁 | 检查什么 | 不通过时怎么处理 |
| --- | --- | --- |
| 生成机制门 | 是否只迁移抽象机制，而非角色、设定、桥段顺序、异常规则、翻牌路径 | 回到样本证据，改写成更高层机制 |
| 正文质量门 | 是否存在逻辑不连贯、剧情拼凑、人物前后不一致、无关描写占比过高、空洞升华、重复灌水 | 退回重构章节运动和场景目的 |
| 平台合规门 | 是否触发异常发文、抄袭洗稿、低俗违规、AI标识/披露、首发独家和合同边界风险 | 停止发布，查平台规则/合同/编辑口径 |

## 6. 对当前项目的行动建议

### 6.1 继续以番茄为第一训练场

理由：

1. 官方规则和福利资料透明度高。
2. 签约、推荐、收益、读完率/追读率路径清楚。
3. 适合先验证前10章和8万字前事件链。
4. 短剧/漫画/有声/IP通道与我们当前异常事件方向有潜在适配。

### 6.2 不把番茄当最终目标

番茄只是当前平台约束。

小说生产运行包最终要能回答：

1. 如果投番茄，怎么写前2万字和8万字验证段。
2. 如果投七猫，怎么准备前2万字+大纲+编辑卖点。
3. 如果投起点，怎么处理强开局、内投/直发材料、追读、上架节奏和长期付费主线。

### 6.3 P2 开始前必须完成的最小补充

进入拆书前，必须确认：

1. P2证据卡已经加入平台字段。
2. 番茄样本池每本都有采样日期和来源。
3. 每本样本标记它更像哪种收益适配方向。
4. 每条机制归纳都能避开合规和相似风险。
5. 后续至少准备七猫/起点少量对照样本，不让番茄样本垄断结论。
6. AI辅助流程必须有“人工主创确认”和“低质AI风险检查”，不能把AI生成正文直接作为可发布稿。

## 7. 当前一句话策略

```text
先用番茄样本验证免费阅读/算法推荐下的强开局和前10章追读机制，再用七猫内投逻辑和起点付费逻辑校准它是不是一种可迁移的小说能力。
```
