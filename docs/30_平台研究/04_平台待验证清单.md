# 平台待验证清单

## 0. 为什么要有这个清单

公开资料只能解决一部分问题。

真正影响投稿和收益的内容，往往在：

1. 作者后台。
2. 当前合同。
3. 编辑口径。
4. 作者真实反馈。
5. 平台当年活动细则。
6. 违规/断更后的实际处理。

所以本清单不追求一次性答完，而是作为后续调研任务池。

## 5. 后续调研方法

### 5.1 来源分级

| 来源 | 用途 |
| --- | --- |
| 官方规则/公告 | 可进入平台机制文档 |
| 作者后台截图 | 可补具体金额和操作路径，但需记录日期 |
| 签约合同样本 | 可补权利义务，但不得公开敏感内容 |
| 编辑口径 | 可作为投稿策略参考，需标注编辑和时间 |
| 作者论坛/社媒反馈 | 只能作为案例和风险线索 |
| 单个作者经验 | 不能直接归纳为平台规则 |

### 5.2 记录格式

每条新资料都按这个格式登记：

```yaml
platform:
topic:
source_type:
source_url_or_note:
sampled_at:
claim:
evidence_summary:
confidence:
decision_impact:
needs_recheck:
```

### 5.3 进入运行包的门槛

平台规则进入运行包前，必须满足：

1. 至少一个官方来源，或两个以上独立高可信来源。
2. 采样日期明确。
3. 能说明它影响哪一个生产决策。
4. 没有和其他来源明显冲突。
5. 如果冲突，必须保留冲突说明，不能强行归纳。
