# 证据提取与归纳规格

## 定位

这是当前番茄小说研究的执行规格。前面的文档解释“为什么这样做”，这个文档规定“实际怎么做”。

后续所有样本研究按这个顺序执行：

```text
样本登记
-> 合规与相似风险边界标注
-> 前 10 章证据抽取
-> 路线候选判断
-> 单书 source-open 备忘
-> route 横向归纳
-> runtime skill 草案
-> clean-room 前 3 章/前 10 章计划
-> 盲审回修
```

## 样本处理分级

不要一上来就对 30 本书全量深拆。分三级。

| 等级 | 范围 | 目的 | 产出 |
| --- | --- | --- | --- |
| L0 样本登记 | 20-30 本 | 建立赛道样本池，排除明显噪声 | `sample_metadata` |
| L1 前 10 章快拆 | 10-15 本 | 判断点击承诺和前 10 章留存机制 | `promise_evidence`、`chapter_motion_evidence`、`opposition_evidence` |
| L2 深拆 | 3-5 本 | 判断事件链、翻牌、长期可连载性 | 单书 source-open 备忘 |
| L3 核心对标 | 1-2 本 | 建 runtime skill 和 clean-room 复刻 | route 归纳、skill 草案 |

如果 L1 都无法证明方向成立，不进入 L2。

评论、书评和章评可以作为 L1/L2 的辅助证据，但它们只验证读者反应，不提供可迁移内容。

允许用评论判断：

1. 读者实际爽点。
2. 读者弃点。
3. 读者看不懂或误解的位置。
4. 读者追更理由。
5. 相似、融梗、AI味、水文等软风险信号。

禁止用评论迁移：

1. 读者提出的剧情建议。
2. 评论区脑洞设定。
3. 单条高赞评论的具体表达。
4. 评论里点名的其他作品桥段。

## 第一批必须提取的证据

### E0 sample_metadata

用途：确认样本可靠性，而不是直接分析剧情。

必须提取：

| 字段 | 说明 |
| --- | --- |
| book_id | 样本编号 |
| title | 书名 |
| author | 作者 |
| platform | 平台 |
| source | 榜单/搜索/推荐来源 |
| sampled_at | 采样日期 |
| status | 连载/完结/断更 |
| word_count | 字数或估算字数 |
| tags | 平台标签 |
| intro_promise | 简介直接承诺 |
| available_scope | 可读章节范围 |
| sample_risk | 是否可能是噪声样本 |

不能写：

```text
这本应该挺火。
这本很像我们想写的。
```

### E1 promise_evidence

用途：判断读者承诺如何建立和首次兑现。

必须提取：

| 字段 | 说明 |
| --- | --- |
| book_id | 样本编号 |
| location | 书名/简介/第几章 |
| promise_text_summary | 承诺内容摘要 |
| promise_type | 爽、悬疑、成长、身份、压迫、智斗、关系、世界观 |
| carrier | 承诺载体：危机、能力、身份、案件、组织、关系 |
| first_payoff_chapter | 第一次兑现章节 |
| payoff_action | 兑现时发生了什么动作 |
| payoff_result | 兑现后读者获得什么满足 |
| evidence_ref | 可回到的章节/位置 |
| transfer_boundary | 可迁移机制、类型公约需重构、仅观察、禁止迁移、高风险复核 |
| risk_note | 相似风险、洗稿风险、平台/法律风险说明 |

有效证据：

```text
第 3 章主角第一次利用异常规则反杀，不是直接战力碾压；这兑现了简介里的“看破异常”的承诺。
```

无效证据：

```text
这本是智斗爽文。
```

### E2 chapter_motion_evidence

用途：判断每章有没有追读推进。

前 10 章逐章提取：

| 字段 | 说明 |
| --- | --- |
| chapter | 章节 |
| start_state | 本章开局处境 |
| trigger | 本章触发事件 |
| protagonist_goal | 主角本章目标 |
| conflict | 本章冲突 |
| new_information | 新信息 |
| payoff | 本章兑现 |
| end_hook | 章末钩子 |
| changed_state | 本章结束后处境变化 |
| transfer_boundary | 本章机制能否迁移 |
| risk_note | 是否存在桥段顺序或高识别内容形状风险 |

有效证据必须能回答：

```text
这一章结束后，主角处境和上一章相比变了吗？
```

如果没变，这章大概率只是水或说明。

### E3 plot_causality_evidence

用途：判断情节是否硬接。

只抽关键场景，不必逐段全抽。

| 字段 | 说明 |
| --- | --- |
| scene | 场景/章节 |
| previous_consequence | 上一幕造成的后果 |
| forced_action | 主角被迫采取的行动 |
| why_now | 为什么现在必须做 |
| obstacle | 阻碍是什么 |
| result | 行动结果 |
| cost | 代价 |
| next_pressure | 如何逼出下一幕 |
| removable | 删除后是否影响后文 |
| transfer_boundary | 因果机制能否迁移 |
| risk_note | 是否过度贴近原书事件链 |

能进入归纳的证据必须具备因果链：

```text
因为 A 已经发生，所以主角不能不做 B；做 B 导致 C；C 逼出 D。
```

### E4 opposition_evidence

用途：判断阻力是否真的会反咬。

| 字段 | 说明 |
| --- | --- |
| opposition_type | 人物、组织、规则、信息差、资源、代价、关系、主角盲区 |
| opposition_goal | 阻力想要什么 |
| leverage | 阻力筹码 |
| active_move | 阻力主动做了什么 |
| protagonist_response | 主角如何反制 |
| cost_to_protagonist | 主角代价 |
| backlash | 阻力是否反咬 |
| escalation | 是否升级 |
| repeatability | 是否能持续生产剧情 |
| transfer_boundary | 阻力机制能否迁移 |
| risk_note | 是否过度贴近原书组织、反派、规则或怪物 |

不能只写“反派很强”。要写反派或规则如何改变主角处境。

### E5 reveal_flip_evidence

用途：判断信息差和翻牌是否推动追读。

| 字段 | 说明 |
| --- | --- |
| open_question | 打开的疑问 |
| hidden_info | 隐藏的信息 |
| clue | 提前给出的线索 |
| false_explanation | 临时误导解释 |
| reveal_chapter | 揭示章节 |
| flip_effect | 翻牌后局势变化 |
| payoff | 回收了什么 |
| next_question | 翻牌后打开的新问题 |
| transfer_boundary | 翻牌机制能否迁移 |
| risk_note | 是否过度贴近原书伏笔、反转和名场面顺序 |

有效翻牌必须改变局势。只解释过去、不推动未来的翻牌不能作为高价值证据。

### E6 support_function_evidence

用途：判断设定是否在服务剧情。

| 字段 | 说明 |
| --- | --- |
| support_item | 能力、组织、民俗、地名、案件、道具、传说 |
| first_appearance | 首次出现位置 |
| function | 制造行动、限制、线索、误导、代价、视觉场面 |
| plot_dependency | 后文是否依赖它 |
| self_indulgence_risk | 是否只是设定展示 |
| transfer_boundary | 支撑物功能能否迁移 |
| risk_note | 是否涉及专属能力、独特怪物、组织体系、地域民俗高识别组合 |

这个提取器只在 L2 深拆时强制使用。

### E7 renderer_evidence

用途：判断叙述执行方式。

| 字段 | 说明 |
| --- | --- |
| scene_type | 爽点、悬疑、战斗、转场、章末 |
| pov_distance | 叙述距离 |
| dialogue_ratio | 对白比例 |
| action_density | 动作密度 |
| exposition_density | 说明密度 |
| inner_monologue_density | 心理密度 |
| ending_breath | 章末气口 |
| transfer_boundary | 文风变量能否迁移 |
| risk_note | 是否过度贴近某作者高识别语言皮肤 |

这个提取器只在要写正文或模仿具体作者时启用。立项阶段不优先。

### E8 adaptation_evidence

用途：判断短剧/漫画/AI 漫改潜力。

| 字段 | 说明 |
| --- | --- |
| visual_scene | 强视觉场面 |
| character_tag | 人设标签 |
| anomaly_visual | 异常/能力可视化 |
| episode_hook | 短剧单集钩子 |
| production_risk | 改编成本或难度 |
| shareable_moment | 是否有可传播名场面 |
| transfer_boundary | 视觉功能能否迁移 |
| risk_note | 是否过度贴近样本名场面、怪物视觉或能力表现 |

立项阶段只做粗筛，不能让改编潜力压倒小说追读。

### E9 reader_feedback_evidence

用途：判断读者真实反应是否支持前面的正文证据。

| 字段 | 说明 |
| --- | --- |
| feedback_scope | 书评、章评、评分页、推书帖、论坛讨论 |
| feedback_sample_size | 粗略样本量 |
| praised_points | 读者反复夸什么 |
| drop_points | 读者反复吐槽或弃书点 |
| confusion_points | 读者看不懂或争议点 |
| chase_reason | 读者为什么追下一章 |
| similarity_alarm | 是否有人说像某本书、融梗、AI味、水文 |
| related_text_evidence | 对应哪条 E1-E8 正文证据 |
| transfer_boundary | 反馈能否进入归纳 |
| risk_note | 软风险或偏样本风险 |

E9 只能辅助验证，不能单独支撑 route。没有对应正文证据时，E9 只能标记为 `observe_only`。

## 归纳到底归纳什么

### G1 单书 source-open 备忘

每本 L2 深拆样本必须输出一份单书备忘。

固定结构：

```text
样本范围：
实际阅读章节：
点击承诺：
前 10 章运动：
第一事件链：
阻力系统：
翻牌系统：
能力/设定功能：
叙述执行：
改编潜力：
主要失败风险：
是否进入 route 归纳：
```

单书备忘禁止只写评价，必须引用证据编号。

### G2 route 横向归纳

至少 3 本同 route 样本才能归纳 route。

固定结构：

| 归纳项 | 必须回答 |
| --- | --- |
| route_name | 这条路线叫什么 |
| included_samples | 哪些样本支持 |
| excluded_samples | 哪些相似样本被排除，为什么 |
| dominant_law | 这条 route 最优先保护什么效果 |
| first_10_pattern | 前 10 章共同结构 |
| causality_pattern | 情节推进共同因果 |
| opposition_pattern | 阻力通常来自哪里 |
| reveal_pattern | 信息如何藏、放、翻 |
| support_pattern | 设定和素材承担什么功能 |
| adaptation_pattern | 哪些元素适合改编 |
| newbie_feasibility | 新人是否能执行 |
| drift_alarms | 写偏的信号 |
| counterexamples | 反例或边界 |

### G2.5 route_candidate 路线候选判断

P2 后先做 route_candidate，不直接升级为 route_model。

固定结构：

| 归纳项 | 必须回答 |
| --- | --- |
| route_candidate_name | 候选路线叫什么 |
| supporting_samples | 哪些样本支持 |
| evidence_scope | 只基于前 10 章、前 30 章还是完整卷 |
| transferable_mechanisms | 哪些机制通过迁移边界检查 |
| observe_only_elements | 哪些元素只能观察，不能迁移 |
| prohibited_shapes | 哪些高识别内容形状必须避开 |
| newbie_feasibility | 新人是否能执行 |
| long_serial_risk | 是否能支撑长篇 |
| similarity_risk | 硬风险/软风险初筛 |
| decision | 进入 L2 深拆、仅保留观察、还是放弃 |

### G3 runtime skill 草案

只有 G2 通过后，才写 runtime skill。

固定槽位：

```text
Activation
Dominant Law
Reader Promise Engine
Plot Causality Engine
Opposition Engine
Reveal / Flip Engine
Serial Pacing Engine
Support Function
Narrative Renderer
Adaptation Engine
Strength Knobs
Transfer Boundary
Drift Alarms
Validation Gate
Evidence Index
```

每个槽位都必须至少有一个证据来源。没有证据来源的规则只能写入 `hypothesis`，不能写入 runtime。

## 晋级门槛

| 晋级 | 条件 |
| --- | --- |
| L0 -> L1 | 样本来源、标签、简介和采样日期清楚 |
| L1 -> L2 | 前 10 章至少有一次明确兑现，且因果不断 |
| L1 -> P2.5 | 每条关键证据都有 transfer_boundary 和 risk_note |
| P2.5 -> L2 | 候选路线通过相似风险初筛，且值得深拆 |
| L2 -> route | 至少 3 本样本出现同类机制，且不依赖禁止迁移内容形状 |
| route -> skill | 有稳定模式、反例边界、失败警报 |
| skill -> clean-room | 能写出前 10 章计划和第一卷事件链 |
| clean-room -> 通过 | 盲审能指出具体追读点，而不是只说“设定不错” |

## 当前禁止项

在第一轮研究中禁止做这些事：

1. 禁止直接写正文。
2. 禁止只凭 1 本书判断赛道。
3. 禁止把“读者承诺”写成一句类型标签。
4. 禁止把设定丰富等同于可连载。
5. 禁止把改编潜力当成主要立项理由。
6. 禁止没有证据编号就更新 runtime skill。
7. 禁止把评论区脑洞、剧情建议或高赞表达当成可迁移素材。
