## 研究顺序

1. 先看 [总控路线图](../00_总控/00_当前总控路线图.md)，确认当前先做 P0.5，再做 P2/L1。
2. 按 [合规与相似风险边界](../00_总控/04_合规与相似风险边界.md) 标注迁移边界。
3. 按 [P2 小说证据提取器协议](../20_小说蒸馏方法/04_P2小说证据提取器协议.md) 执行前 10 章证据抽取。
4. 按 [证据提取与归纳规格](02_证据提取与归纳规格.md) 做 L0-L3 分级样本处理。
5. 用样本证据拆单书层：承诺、因果、阻力、翻牌、支撑物、改编信号。
6. 横向归纳 route：同赛道不同路线的共同结构、反例边界和失败模式。
7. 只有 route 归纳通过，才写 runtime book/genre skill。
8. 最后 clean-room 生成前 3 章/前 10 章计划，并用盲审回修。

## 关键原则

不要把“题材热门”直接等同于“我们能写”。番茄立项要同时满足三件事：读者点得进、前 10 章留得住、后续能稳定连载。

所有进入 runtime skill 的判断，都必须先过 [批判性审查门](../00_总控/02_批判性审查门.md)：能定位证据、能解释反例、能 clean-room 验证，才允许升级为生成规则。
