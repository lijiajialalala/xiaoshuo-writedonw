# 前期规划目录优化建议

更新时间：2026-05-09

## 0. 当前判断

`D:\小说创作\前期规划` 是早期规划目录，适合沉淀平台对比、规则风险、流程方案和协议模板。

但现在项目已经进入下一阶段：

- 番茄方向基本收敛
- 都市高武 / 异常事件方向开始成型
- 文案蒸馏方法准备迁移到小说
- 后续会有样本池、作者研究、单书项目、章节生成和验证闭环

所以原来的 `前期规划` 目录已经不适合作为主工作区。

## 1. 为什么需要整体优化

原目录的问题不是文件放错，而是阶段变了。

早期目录按“规划主题”组织：

- 平台对比
- 规则风险
- 创作流程
- 系统设计
- 协议模板
- 来源索引

这个结构适合做决策前研究。

但后续真正写小说时，工作对象会变成：

- 平台机制
- 赛道样本
- 作者 / 书籍蒸馏
- 单书 bible
- 章节状态
- 复盘验证

这些不应该都继续塞进 `前期规划`。

## 2. 建议的新主结构

后续仓库建议逐步演进为：

```text
docs/
  前期规划/                 # 早期规划归档，保留原始判断脉络
  平台研究/                 # 番茄机制、收益、榜单、推荐、改编
  小说蒸馏/                 # 方法论、schema、抽取模板、runtime slots
  赛道研究/                 # 都市高武、异常事件、749/收容等赛道样本
  项目方案/                 # 具体书的立项、story bible、卷纲、章节策略
research/
  samples/                  # 小样本结构表，不放大体量全文
  route_comparison/          # route 横向对比
schemas/
  novel_sample_record.yaml
  book_promise_record.yaml
  chapter_card.yaml
  scene_card.yaml
.agents/skills/
  novel-distiller/
  fanqie-platform/
  target-author-runtime/
```

## 3. 当前迁移策略

不建议现在大规模重排所有旧文件。

更稳的做法是：

- 先把 `前期规划` 整体放进仓库，保留原始上下文。
- 新工作不再继续往 `前期规划` 根目录塞。
- 新增内容按“平台研究 / 小说蒸馏 / 赛道研究 / 项目方案”逐步拆出去。
- 等新结构稳定后，再把旧目录中的文件分批迁移，并在 README 里保留索引。

## 4. 立即要做的下一步

建议下一轮先建三个目录：

- `docs/小说蒸馏`
- `docs/赛道研究/番茄都市高武`
- `docs/平台研究/番茄`

然后把已有新文件按这个方向逐步迁过去：

- 小说蒸馏迁移思考 -> `docs/小说蒸馏`
- 番茄平台立项双目标模型 -> `docs/平台研究/番茄`
- 番茄单书层生存模型 -> `docs/平台研究/番茄`
- 番茄都市高武榜单样本结构分析 -> `docs/赛道研究/番茄都市高武`

这一步完成后，`docs/前期规划` 就可以视为历史规划区，而不是主工作区。
