## 更新规则

1. 平台机制、收益机制、合同强度、AI 风险，放入 `docs/番茄专项研究/`。
2. 样本拆解、榜单观察、候选书单，放入 `research/番茄专项/`。
3. 结构模板和协议，优先放入 `schemas/` 或 `docs/前期规划/05_协议模板/`。
4. 如果某个结论会影响立项方向，必须写清楚证据来源和待验证问题。
5. 如果某个判断要升级为生成规则，必须先通过 `docs/番茄专项研究/04_批判性审查门.md`。

## 当前核心问题

我们不是先问“能不能写一个好设定”，而是先问：

1. 番茄当前什么读者承诺更容易获得流量。
2. 都市高武、都市异能、异常事件、749/斩神相邻题材各自靠什么留人。
3. 新人作者能不能执行这个结构，前 10 章是否足够快。
4. 这个方向是否有短剧化、漫画化和视觉传播潜力。
5. 后续能不能稳定生产副本、事件、对手和翻牌。
