# 番茄立项验证闭环

## 目标

在正式写正文前，用样本和平台机制验证一个小说方向是否值得投入。当前候选方向以都市高武、都市异能、异常事件处理、749/斩神相邻题材为主。

## 闭环流程

这个流程不是传统拆书表。它要从 `douyin-wenan` 的作者蒸馏流水线迁移过来，只是把短视频的“单篇文案”换成小说的“整书/卷/章节/场景”多颗粒度证据。

```text
平台机制研究
-> 榜单/样本采集
-> 样本文本质量确认
-> API 结构证据抽取
-> source-open 多颗粒度通读归纳
-> 单书生成模型 / 叙述渲染器拆分
-> 题材 route selector
-> runtime book/genre skill
-> clean-room 开局复刻
-> blind-review / 数据化评审
-> 回到样本证据修 skill
```

具体步骤：

1. 平台机制确认：先明确番茄当前更奖励什么行为，尤其是点击、留存、完读、追更和转化。
2. 榜单样本收集：每个候选方向先收 20-30 本，记录来源、日期、榜单、标签、简介和章节范围。
3. 结构证据抽取：用固定 schema 抽读者承诺、事件因果、对手阻力、揭示翻牌、章节钩子、改编潜力。
4. source-open 通读归纳：不能只看抽取表；至少通读前 30 章和一个完整事件卷，记录整段运动。
5. route selector：判断同赛道内部是否存在不同路线，例如战力升级线、异常调查线、官方组织线、民俗禁区线。
6. runtime skill：把归纳结果写成可执行生成协议，而不是分析报告。
7. clean-room 复刻：不用原书表达，按 skill 生成一个新开局或第一卷设计。
8. 盲审回修：用“像不像番茄可跑书”“有没有追读”“是否自嗨”“是否可连载”来回修模型。

## 单书层需要回答的问题

这些不是主方法，只是小说 runtime slot。它们要由证据抽取和通读归纳共同填充，不能人工看几章凭感觉填表。

1. 这本书最早向读者承诺什么：爽、悬疑、翻牌、关系拉扯、成长、压迫感、智斗，还是生存压力。
2. 这个承诺是不是单一的：长篇小说可以有多个承诺，但必须有主承诺和副承诺。
3. 前 3 章承诺是否清晰，前 10 章是否兑现过一次。
4. 主角能力是直接变强，还是通过识破规则、找到破绽、反制对手来制造爽感。
5. 对手和阻力是否会反咬，还是只等主角输出。
6. 信息是否有隐藏和释放节奏，是否存在“读者知道主角能赢，但不知道怎么赢”的追读动力。
7. 第一卷是否能形成完整事件闭环，而不是只有设定展示。

## 作者层需要回答的问题

作者层暂时不是第一优先级。番茄立项阶段优先抽“赛道/单书模型”，等我们确认某个作者连续多本或同一本长线表现值得学习，再建立作者层 runtime skill。

1. 这个作者通常如何建立读者承诺。
2. 他如何兑现承诺：靠打脸、升级、破局、反转、情绪补偿、关系推进还是世界观展开。
3. 他什么时候升级承诺：从个人危机到组织危机，从小副本到大规则，从能力展示到身份揭露。
4. 他什么时候转移承诺：爽点疲劳后如何切到悬疑、关系、世界观或更强对手。
5. 他如何处理节奏低谷：插入新危机、翻牌、反咬、误判、代价还是新人物。

## 多颗粒度拆解

小说不能用短视频的“整篇”直接替代。小说的 source-open gate 要分层：

整书层用于判断商业承诺、长期赛道、主线驱动和完结/长线风险。

卷/副本层用于判断一个事件链如何开始、升级、反转、收束，并为下一卷留钩。

章节层用于判断番茄追读节奏：每章目标、冲突、钩子、爽点、悬疑点。

场景层用于判断具体执行：谁想要什么、谁阻止、代价是什么、信息如何释放。

段落层用于判断叙述渲染器：句子长度、对白比例、动作密度、心理活动、转场气口。

最低通读要求：

1. 所有样本至少读书名、简介、标签、前 10 章。
2. 候选深拆样本至少读前 30 章。
3. 核心对标样本至少读一个完整卷/副本。
4. 如果要升级为 runtime skill，必须读完整书或足够覆盖开局、中段、后期的代表章节。

## 暂定评分维度

| 维度 | 问题 | 分值 |
| --- | --- | --- |
| 点击承诺 | 书名、简介、标签是否一眼能懂 | 1-5 |
| 前 3 章留存 | 是否快速进入冲突和能力展示 | 1-5 |
| 前 10 章兑现 | 是否完成一次爽点或悬疑兑现 | 1-5 |
| 事件链能力 | 是否能持续生产副本和异常事件 | 1-5 |
| 对手阻力 | 对手、规则、误判、代价是否会反咬 | 1-5 |
| 翻牌设计 | 是否能做到“知道会赢，但不知道怎么赢” | 1-5 |
| 新人可执行 | 是否依赖极强笔力或资料量 | 1-5 |
| 改编潜力 | 是否适合漫画、短剧、强视觉名场面 | 1-5 |

## 当前判断

都市高武不能只理解成“战力升级”。如果我们要做异常事件处理方向，更适合把高武当作推动案件和危机解决的能力系统，而不是单纯比大小。重点应放在事件、规则、破绽、反制和翻牌上。

这个方向需要警惕一个风险：体系太杂会变成设定展览。修仙、异能、巫蛊、武侠可以共存，但必须由统一的世界规则管理，否则读者只会看到作者在硬塞素材。
