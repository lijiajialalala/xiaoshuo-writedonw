# 当前总控路线图

## 当前阶段

当前阶段不是写正文，也不是完整搭建小说 Agent。当前阶段只有一个目标：

```text
用最小证据闭环，判断“番茄都市高武/异常事件方向”是否值得进入立项。
```

如果这个闭环跑不通，后面做再多 skill、schema、流程都是空转。

## 当前必须回答的 4 个问题

1. 番茄是否仍然是当前最优先平台。
2. 都市高武/异常事件方向是否真实有样本支撑，而不是我们主观觉得好玩。
3. 这个方向的前 10 章留人机制是什么。
4. 我们能否 clean-room 写出一个不自嗨、有追读的前 3 章/前 10 章计划。

暂时不回答：

1. 具体主角最终叫什么。
2. 百万字大纲怎么排。
3. 完整世界观怎么设计。
4. 是否马上建立完整作者 skill。
5. 是否马上写正文。

## 单一主流程

后续只按这条流程推进：

```text
P0 假设登记
-> P1 L0 样本池
-> P2 L1 前 10 章快拆
-> P3 L2 深拆
-> P4 route 归纳
-> P5 runtime skill 草案
-> P6 clean-room 开局测试
-> P7 盲审回修
-> P8 立项或放弃
```

## 阶段定义

| 阶段 | 要做什么 | 产出 | 通过条件 |
| --- | --- | --- | --- |
| P0 假设登记 | 把平台、题材、改编、AI 风险假设列出来 | 假设清单 | 每个假设有证据需求或待验证状态 |
| P1 L0 样本池 | 收 20-30 本番茄同赛道书 | 样本登记表 | 来源、标签、简介、采样日期清楚 |
| P2 L1 快拆 | 对 10-15 本做前 10 章快拆 | 前 10 章证据表 | 能看出承诺、兑现、因果、阻力 |
| P3 L2 深拆 | 对 3-5 本读前 30 章或完整事件卷 | 单书 source-open 备忘 | 能解释事件链和翻牌系统 |
| P4 route 归纳 | 横向归纳 2-4 条 route | route 归纳表 | 每条 route 至少 3 个样本或明确标为假设 |
| P5 skill 草案 | 把最强 route 写成 runtime 生成协议 | runtime skill draft | 每条规则有证据编号 |
| P6 clean-room 测试 | 写前 3 章和前 10 章计划 | 新开局测试稿 | 不搬运原文，承诺和因果能跑 |
| P7 盲审回修 | 做追读/自嗨/番茄感审查 | 盲审记录 | 问题能回到证据或 skill 槽位 |
| P8 立项决策 | 决定写、改方向、还是放弃 | 立项结论 | 有明确继续/停止理由 |

## 当前下一步

下一步不是补方法论，而是做 P0 和 P1。

### P0 假设登记

先登记这些假设：

| 假设 | 当前状态 | 需要什么证据 |
| --- | --- | --- |
| 番茄是当前最优先平台 | 待复核 | 平台规则、收益机制、AI 风险、合同强度 |
| 都市高武/异常事件有流量空间 | 待验证 | 榜单样本、标签样本、近期热书 |
| 异常事件比纯战力升级更适合我们 | 待验证 | 前 10 章结构、事件链可生产性 |
| 多能力体系能增加新鲜感 | 高风险假设 | 统一规则、避免设定拼盘的样本证据 |
| 方向要兼顾短剧/漫画改编 | 次级假设 | 视觉场景、角色标签、单集钩子 |

### P1 L0 样本池

先收 20-30 本，不做深拆。

每本只记录：

```text
书名
作者
平台
来源
采样日期
状态
字数
标签
简介承诺
为什么入选
可能偏差
```

## 暂停事项

在 P2 没完成前，暂停：

1. 写正文。
2. 设计完整世界观。
3. 创建作者级 skill。
4. 讨论最终主角名和细节人设。
5. 把多能力体系当默认方案。
6. 直接规划百万字结构。

## 总控判断

目前项目最大风险不是“不够复杂”，而是太早进入复杂。

正确顺序应该是：

```text
先证明这个方向能留人
再证明我们能写
再证明它能长期连载
最后再扩展成完整创作系统
```

如果顺序反了，项目会变成设定和方法论都很丰富，但没有一本能开写的书。
