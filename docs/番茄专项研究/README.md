# 番茄专项研究

这个目录只解决一个问题：在番茄上写什么、怎么写、为什么这个方向更可能跑出来。

当前不直接进入正文。先把番茄平台机制、收益机制、AI 风险、题材榜单、读者承诺、开局留存和后续改编潜力拆清楚。

## 研究对象

1. 平台机制：推荐、完读、追更、互动、书名简介、标签和冷启动。
2. 收益机制：签约、分成、保底、完结奖励、听书/漫画/短剧等衍生收益。
3. AI 风险：平台规则、内容雷区、相似性风险、合同责任、审核风险。
4. 题材方向：都市高武、都市异能、异常事件、749/斩神相邻、悬疑智斗爽文。
5. 改编潜力：是否适合 AI 漫改、短剧改编、视觉化名场面、单集钩子。

## 研究顺序

1. 先过 [批判性审查门](04_批判性审查门.md)，把“平台/题材/样本”假设标清楚。
2. 按 [证据提取与归纳规格](05_证据提取与归纳规格.md) 做 L0-L3 分级样本处理。
3. 用样本证据拆单书层：承诺、因果、阻力、翻牌、支撑物、改编信号。
4. 横向归纳 route：同赛道不同路线的共同结构、反例边界和失败模式。
5. 只有 route 归纳通过，才写 runtime book/genre skill。
6. 最后 clean-room 生成前 3 章/前 10 章计划，并用盲审回修。

## 关键原则

不要把“题材热门”直接等同于“我们能写”。番茄立项要同时满足三件事：读者点得进、前 10 章留得住、后续能稳定连载。

所有进入 runtime skill 的判断，都必须先过 [批判性审查门](04_批判性审查门.md)：能定位证据、能解释反例、能 clean-room 验证，才允许升级为生成规则。

当前真正可执行的主文档是 [05_证据提取与归纳规格.md](05_证据提取与归纳规格.md)。`01-04` 是背景、方法和审查约束，不应替代执行规格。
