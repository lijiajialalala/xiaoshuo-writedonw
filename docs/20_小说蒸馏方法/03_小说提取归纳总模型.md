# 小说提取归纳总模型

## 0. 先纠正范围

我们要做的不是“番茄小说生产运行包”，而是“小说生产运行包”。

番茄只是当前阶段的平台限定词，它提供：

1. 当前样本来源。
2. 当前流量和收益约束。
3. 当前节奏、读者、题材、签约和风控要求。

但最终模型不能只服务番茄。它应该能迁移到其他平台、其他题材、其他作者和其他项目。

所以总目标是：

```text
公开样本
-> 结构证据提取
-> source-open 多颗粒度归纳
-> 内容内核发现
-> 自有 house style
-> 小说生产运行包
-> clean-room 写作
-> 盲审和回修
```

## 0.1 从文案系统学到的新原则

文案项目最新的关键修正是：

```text
对标样本不是目标，而是显微镜。
复刻不是生产，而是诊断工具。
真正目标是：从样本中发现可迁移的内容内核，再用我们自己的 house style 生产。
```

迁移到小说后，这句话要改成：

```text
对标书不是目标，而是显微镜。
开局复刻不是生产，而是诊断工具。
真正目标是：从多本样本中发现可迁移的小说内容内核，把内核适配到当前平台和题材，再用我们自己的 Story Bible / Style Bible 生产。
```

这解决一个很重要的问题：

```text
不能直借一本书，否则容易变成换皮/洗稿。
也不能完全不借鉴，否则容易变成低质量 AI 水文。
我们要借的是抽象生成机制，不是高识别内容形状。
```

所以小说系统必须保留两件事：

1. 保留旧模式的锋利反馈：用 clean-room 开局和盲审暴露模型缺口。
2. 保留新模式的安全边界：不把某本书、某作者、某个爆款开局当成生产目标。

小说版完整链路应理解为：

```text
样本研究
-> 受控 clean-room 诊断
-> 盲审/追读诊断
-> 正文身体/叙事身体抽取
-> 小说内容内核发现
-> 平台/赛道适配
-> 自有 Story Bible + Style Bible
-> 独立生产
-> 评估与回写
```

这里的 `正文身体/叙事身体` 指的不是文笔，而是小说真实移动的方式：

```text
这一章怎么进入
主角什么时候被迫行动
第一个阻力什么时候反咬
线索什么时候出现
翻牌什么时候改变局势
爽点什么时候兑现
章末钩子怎么把读者推到下一章
哪些不均匀、拖延、重复、误导是样本真实追读结构的一部分
```

这里的 `内容内核` 指的也不是题材标签，而是这类小说为什么能让读者持续追下去：

```text
异常处置内核：异常出现 -> 错误解释 -> 主角测试规则 -> 代价 -> 破绽 -> 反制 -> 更大异常。
规则读档内核：失败结果 -> 复盘限制 -> 重新选择 -> 信息差变化 -> 局势改写。
都市高武升级内核：压迫处境 -> 能力触发 -> 资源争夺 -> 对手反咬 -> 阶层位置上升。
破绽推演内核：强敌压迫 -> 观察异常 -> 找到漏洞 -> 低成本反杀 -> 暴露更大棋局。
```

这些内核可以迁移；具体角色、组织、能力名称、异常规则、桥段顺序、名场面和高识别文风不能迁移。

## 1. 最终结果到底是什么

最终结果不是一堆拆书表，也不是一份 prompt。

最终结果应该是一套可以支撑长篇小说生产的运行包，至少包含 9 个交付物：

| 交付物 | 作用 |
| --- | --- |
| 01_项目定位书 | 决定这本书写给谁、在哪个平台发、主承诺是什么、禁止写偏到哪里 |
| 02_路线模型库 | 归纳不同小说打法，例如异常处置、规则读档、系统高武、破绽推演 |
| 03_单书生成模型 | 解释一本对标书如何建立承诺、推进前 10 章、闭合第一卷 |
| 04_Story Bible | 我们自己的主角、世界、组织、能力体系、关系、长期悬念 |
| 05_卷/副本/事件链模型 | 第一卷和后续事件如何开局、升级、反咬、翻牌、收束 |
| 06_章节规划器 | 每章开局处境、触发、目标、冲突、兑现、钩子、状态变化 |
| 07_场景生成器 | 单场景里人物目标、阻力动作、信息释放、动作/对白/心理如何落地 |
| 08_叙述渲染器/Style Bible | 文笔、文风、叙述距离、句段节奏、对白气口、动作密度 |
| 09_验证门和回修协议 | 判断前 3 章、前 10 章、第一卷是否能追读、可连载、可改编 |
| 10_内容内核卡 | 记录从样本中发现的可迁移生成机制，以及禁用的高识别内容形状 |
| 11_House Style | 定义我们自己的小说声音、叙述取舍、爽感强度、技术/民俗/高武可信边界 |

这 11 个东西合起来，才叫“小说生产运行包”。

## 2. 这个运行包必须能回答什么

它不是只回答“这本书写什么”，而是要覆盖小说生产的全部关键问题。

### 2.1 商业和读者层

必须回答：

1. 这本书的目标读者是谁。
2. 读者点进来之前看到什么承诺。
3. 前 3 章给读者什么即时满足。
4. 前 10 章必须兑现什么。
5. 这本书靠爽、悬疑、智斗、成长、关系、身份、世界观还是情绪补偿留人。
6. 哪个是主承诺，哪些只是副承诺。
7. 平台约束是什么，哪些写法在这个平台不适合。

### 2.2 大纲和结构层

必须回答：

1. 全书主线是什么。
2. 第一卷解决什么问题。
3. 前 10 章完成什么小闭环。
4. 每个副本/事件为什么开始、如何升级、如何结束。
5. 长线伏笔如何分布。
6. 哪些信息现在不能说，哪些必须尽早兑现。
7. 后续 30 万字、50 万字、100 万字靠什么持续生产剧情。

### 2.3 人物和关系层

必须回答：

1. 主角为什么行动，不行动会失去什么。
2. 主角的核心能力、欲望、短板、盲区是什么。
3. 配角各自承担什么剧情功能。
4. 对手不是“坏”，而是想要什么。
5. 关系如何制造压力、误判、保护、背叛、交易和牵连。
6. 人物变化如何对应剧情推进，而不是单独写成长感悟。

### 2.4 能力体系和设定层

必须回答：

1. 能力体系的底层规则是什么。
2. 主角能力为什么能持续制造剧情，而不是只负责变强。
3. 能力有什么限制、代价、误用风险。
4. 民俗、地域、组织、怪物、异常、道具各自服务什么剧情功能。
5. 设定如何制造行动、限制、线索、误导、翻牌和视觉场面。
6. 哪些设定只是好玩但不该进入正文。

### 2.5 叙事因果层

必须回答：

1. 为什么这一章必须发生。
2. 上一幕后果如何逼出下一幕。
3. 主角为什么不能不行动。
4. 主角赢了以后，为什么压力反而升级。
5. 哪些场景删掉后剧情仍然成立，因此应该删除或重写。

### 2.6 阻力和对手层

必须回答：

1. 阻力来自敌人、组织、规则、信息差、资源、代价、关系还是主角盲区。
2. 阻力手里有什么筹码。
3. 阻力是否会主动行动。
4. 主角反制后付出什么代价。
5. 阻力如何反咬并升级。
6. 这个阻力能否持续生产后续剧情。

### 2.7 揭示和翻牌层

必须回答：

1. 当前打开了什么疑问。
2. 哪些信息读者知道，主角不知道。
3. 哪些信息主角知道，读者不知道。
4. 哪些线索提前埋了。
5. 哪些解释是临时误导。
6. 翻牌发生在哪一章。
7. 翻牌后局势是否改变，而不是只解释过去。
8. 翻牌后打开什么新问题。

### 2.8 文笔和文风层

必须回答：

1. 叙述距离是贴主角、旁观解释，还是全知推进。
2. 一章里动作、对白、心理、说明的大致比例。
3. 爽点段落怎么写，悬疑段落怎么写，战斗段落怎么写，转场怎么写。
4. 句子长短、段落呼吸、章末气口是什么样。
5. 对白是信息交换、冲突推进，还是情绪拉扯。
6. 设定说明如何压缩进行动里。
7. 哪些表达会显得 AI、空泛、装腔、过度文学化。
8. 什么叫“这本书自己的语言稳定性”。

文笔不是最后随便补的皮肤。它是运行包里的核心模块。

区别只是：文笔不应该在 route 未定时最先抽。否则很容易写得像某本书，但不知道这本书为什么能跑。

### 2.9 改编和传播层

必须回答：

1. 哪些场景适合漫画化。
2. 哪些冲突适合短剧单集钩子。
3. 哪些能力、怪物、异常、地标有视觉记忆点。
4. 哪些角色标签一眼能懂。
5. 哪些名场面可以传播。
6. 哪些设定改编成本太高，不适合早期重押。

### 2.10 验证和回修层

必须回答：

1. 前 3 章读者是否愿意继续。
2. 前 10 章是否兑现了点击承诺。
3. 第一卷是否形成完整事件闭环。
4. 哪一章开始拖。
5. 哪个场景是硬接。
6. 哪个阻力没有反咬。
7. 哪个翻牌没有改变局势。
8. 哪些语言问题导致 AI 感或阅读阻力。
9. 问题应该回修到大纲、人物、能力体系、章节、场景还是文风。

## 3. route 模型到底是什么

`route` 不是玄学词。中文可以叫“路线模型”或“打法模型”。

它指的是：

```text
一类小说在建立读者承诺、推进前 10 章、制造阻力、释放信息、兑现爽点和维持连载时，反复出现的生成路线。
```

它不是题材标签。

比如“都市高武”只是题材标签，但它下面可能有不同路线：

| 题材标签 | 可能的路线模型 |
| --- | --- |
| 都市高武 | 系统升级爽文路线 |
| 都市高武 | 废柴破绽推演路线 |
| 都市高武 | 组织任务/异常处置路线 |
| 都市高武 | 读档试错/规则破解路线 |
| 都市高武 | 修仙大佬碾压异能都市路线 |

同一个题材，路线不同，写法完全不同。

路线模型要回答：

1. 读者为什么点进来。
2. 前 3 章怎么给钩子。
3. 前 10 章怎么兑现。
4. 主角怎么赢。
5. 对手怎么反咬。
6. 能力体系怎么服务剧情。
7. 章末钩子靠什么。
8. 哪些写法会偏离这条路线。

例子：

```text
规则读档路线：
主承诺不是“主角最强”，而是“主角能通过死亡试错找到规则破绽”。
读者追的不是能不能赢，而是怎么赢。
阻力通常来自规则误导、死亡代价、信息不完整、时间限制。
翻牌要频繁改变解法，而不是只给战力奖励。
```

这就是路线模型。

## 4. 提取和归纳的区别

### 4.1 提取是什么

提取是从样本里拿可定位证据。

它回答：

```text
原文在哪里做了这件事？
第几章？
用什么动作做的？
造成什么结果？
能否回到文本核对？
```

提取不能写成：

```text
这本书节奏很好。
主角很爽。
设定很有意思。
```

提取应该写成：

```text
第 3 章，主角第一次利用异常规则反杀，兑现了简介里的“看破异常”承诺。
第 5 章，组织封锁现场导致主角不能报警，只能接受内部任务。
第 8 章，前面出现的镜面延迟线索翻牌为“镜中人预判现实”，主角原计划失效。
```

### 4.2 归纳是什么

归纳是从多条证据里提炼可复用机制。

它回答：

```text
这些证据共同说明什么生成规律？
这个规律只属于这本书，还是同一路线多本书都出现？
这个规律能不能 clean-room 迁移？
迁移时不能复制什么？
```

归纳不能写成：

```text
我们也要写一个镜子怪谈。
```

归纳应该写成：

```text
规则型异常事件通常先给一个可观察异常，再给一个错误解释，随后让主角测试规则并付出代价。翻牌不能只解释规则，而要让原解法失效，逼出下一轮行动。
```

## 5. 小说需要提取哪些层

小说比文案复杂，所以必须分层。不是一开始全抽，也不是只抽前 10 章。

### 5.1 样本元数据层

抽：

1. 平台。
2. 书名。
3. 作者。
4. 标签。
5. 字数和状态。
6. 简介承诺。
7. 榜单或搜索来源。
8. 采样日期。
9. 样本风险。

归纳：

1. 这个样本是否代表赛道。
2. 它是头部、腰部、新书、噪声还是相邻题材。
3. 是否值得进入前 10 章提取。

### 5.2 单书承诺层

抽：

1. 书名/简介/标签承诺。
2. 第一章实际承诺。
3. 前 3 章承诺落点。
4. 前 10 章首次兑现。
5. 承诺是否升级或转移。

归纳：

1. 这本书到底靠什么留人。
2. 主承诺和副承诺是什么。
3. 简介和正文是否一致。
4. 这种承诺能否迁移到我们的项目。

### 5.3 前 10 章结构层

抽：

1. 每章开局处境。
2. 触发事件。
3. 主角目标。
4. 本章阻力。
5. 本章行动。
6. 本章兑现。
7. 章末钩子。
8. 结束后状态变化。

归纳：

1. 前 10 章有没有形成追读链。
2. 哪几章兑现，哪几章加压。
3. 有没有连续多章只铺设定。
4. 新人写这个节奏是否可执行。

### 5.4 卷/副本/事件链层

抽：

1. 事件如何开始。
2. 事件表层目标。
3. 中途误导。
4. 阻力升级。
5. 代价出现。
6. 核心翻牌。
7. 事件闭合。
8. 事件结束后留下的新问题。

归纳：

1. 第一卷如何闭合。
2. 一个副本/案件/事件需要多少章。
3. 事件链能否批量生产。
4. 每卷如何连接下一卷。

### 5.5 人物层

抽：

1. 主角初始欲望。
2. 主角外部目标。
3. 主角内部短板。
4. 主角能力获取方式。
5. 主角每次选择的代价。
6. 配角出场功能。
7. 对手目标和筹码。
8. 关系变化节点。

归纳：

1. 主角行动是否稳定。
2. 人物不是标签，而是能否持续制造选择。
3. 配角是否只是陪衬。
4. 对手是否能持续推动剧情。

### 5.6 能力体系和设定层

抽：

1. 能力底层规则。
2. 能力使用条件。
3. 能力限制。
4. 能力代价。
5. 能力升级方式。
6. 能力如何解决问题。
7. 能力如何制造新问题。
8. 设定首次出现位置和剧情功能。

归纳：

1. 能力体系是否统一。
2. 能力是否只负责变强，还是能生产剧情。
3. 设定是否服务行动、限制、线索、误导、翻牌。
4. 多能力体系是否会变成拼盘。

### 5.7 阻力系统层

抽：

1. 阻力类型。
2. 阻力目标。
3. 阻力筹码。
4. 阻力主动动作。
5. 主角反制。
6. 主角代价。
7. 阻力反咬。
8. 阻力升级。

归纳：

1. 主角是否单向输出。
2. 阻力是否能持续生产剧情。
3. 哪些阻力适合前 10 章，哪些适合第一卷后半。
4. 对手/规则/组织的层级如何安排。

### 5.8 揭示/翻牌层

抽：

1. 打开的疑问。
2. 隐藏信息。
3. 线索。
4. 误导解释。
5. 揭示章节。
6. 翻牌效果。
7. 局势变化。
8. 新问题。

归纳：

1. 信息什么时候藏。
2. 线索怎么给才不突兀。
3. 翻牌如何改变局势。
4. 翻牌密度是否适合平台节奏。

### 5.9 文笔/文风/叙述渲染层

抽：

1. 叙述人称。
2. 叙述距离。
3. 句子平均长度。
4. 段落长度。
5. 对白比例。
6. 动作密度。
7. 心理描写密度。
8. 设定说明方式。
9. 爽点段落写法。
10. 悬疑段落写法。
11. 战斗段落写法。
12. 章末钩子落句。
13. 常用转场方式。
14. AI 感风险表达。

归纳：

1. 这本书读起来快还是慢。
2. 语言是干脆、轻松、压迫、悬疑、热血还是口语化。
3. 哪些句式构成稳定文风。
4. 哪些表达不能迁移，否则会像抄文风。
5. 我们自己的 style bible 应该采用哪些可迁移规则。

### 5.10 改编潜力层

抽：

1. 强视觉场景。
2. 怪物/异常/能力可视化。
3. 角色标签。
4. 单集钩子。
5. 名场面。
6. 改编成本风险。

归纳：

1. 哪些元素适合漫画。
2. 哪些元素适合短剧。
3. 哪些元素适合 AI 漫改。
4. 改编潜力是否服务追读，而不是喧宾夺主。

## 6. 分阶段路径

不能一开始把所有层全抽，否则工作量会爆炸。

### P1：样本池

目标：

```text
知道有哪些样本，哪些值得继续看。
```

抽：

```text
样本元数据
简介承诺
标签
样本风险
```

不抽：

```text
文风
完整大纲
人物生命周期
长期能力体系
```

### P2：前 10 章结构证据

目标：

```text
判断这本书前 10 章如何留人。
```

抽：

```text
读者承诺
章节运动
叙事因果
阻力反咬
揭示/翻牌
能力/设定功能
```

暂不重点抽：

```text
完整文风
百万字结构
全人物关系网
```

### P3：第一卷/事件链深拆

目标：

```text
判断一条路线能不能持续连载。
```

抽：

```text
第一卷事件链
副本开合
中层阻力
能力限制
信息释放节奏
```

### P4：人物/能力/长期结构

目标：

```text
判断它如何支撑 30 万字、50 万字、100 万字。
```

抽：

```text
人物成长
关系变化
能力升级
组织层级
长期反派
世界观扩展
```

### P5：文笔/文风/叙述渲染器

目标：

```text
让我们自己的正文长期稳定、可读、低 AI 感。
```

抽：

```text
高爽点段落
高悬疑段落
高冲突对白
战斗段落
章末段落
转场段落
设定说明段落
```

产出：

```text
style_bible
narrative_renderer
anti_ai_expression_rules
scene_type_rendering_rules
```

## 9. 当前最重要的判断

现在不是“不重视文风”，而是不能把文风放在路线选择之前。

正确顺序是：

```text
先确定这本小说靠什么机制活下来
再确定它用什么文风稳定表达
最后把结构模型和文风模型合成运行包
```

如果顺序反了，就会变成：

```text
语言像了
但剧情不知道为什么发生
风格像了
但读者不知道为什么追
文笔顺了
但书没有可连载发动机
```

这就是“得其形，不得其神”在小说里的版本。

## 10. 一句话定义

小说提取归纳模型的目的不是拆懂样本，而是把样本中可验证、可迁移、可生成的机制，归纳成一套能支撑我们从立项、大纲、人物、能力、章节、场景、文风到验证回修的小说生产运行包。
