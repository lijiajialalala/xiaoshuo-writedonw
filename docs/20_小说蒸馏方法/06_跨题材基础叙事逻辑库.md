# 跨题材基础叙事逻辑库

## 0. 一句话定义

跨题材基础叙事逻辑库不是题材库，也不是 route 库。

它记录长篇小说持续可读的“油盐酱醋”：不管是都市高武、悬疑、玄幻、历史、仙侠、民俗、科幻，读者愿意继续看下去，通常都需要某些底层逻辑在工作。

## 1. 为什么要单独建这一层

如果只研究都市高武，容易把都市高武的局部套路误判成小说通用能力。

如果只研究单本爆款，容易把高识别内容形状误判成可迁移机制。

如果只研究文风，容易出现“句子像了，但读者不想看下一章”。

所以基础叙事逻辑应该在 route 之前或与 route 并行抽取：

```text
跨题材基础逻辑
-> 平台差异对照
-> 题材 route 差异
-> 单书机制
-> 文风和渲染器
```

## 2. 它和其他层的区别

| 层级 | 回答什么 | 例子 |
| --- | --- | --- |
| 跨题材基础逻辑 | 长篇为什么持续可读 | 压力、缺口、代价、后果、信息差 |
| 平台约束 | 当前平台如何影响文本 | 三章爆点、直发反馈、签约门槛、更新成本 |
| 题材 | 写什么领域 | 都市高武、民俗悬疑、玄幻升级 |
| route | 这类书如何生成追读 | 异常处置、规则破解、系统升级、身份揭露 |
| 单书机制 | 这本书如何运行 | 某本书的主角、能力、事件链、关系网 |
| 文风渲染 | 语言如何呈现 | 叙述距离、对白密度、心理戏、句段节奏 |

## 3. 第一批候选基础逻辑

| 基础逻辑 | 核心问题 | 提取时看什么 | 生产时影响什么 |
| --- | --- | --- | --- |
| 缺口逻辑 | 主角缺什么，读者等什么被补上 | 开局缺失、身份缺口、能力缺口、情感缺口 | 主承诺和长期追读 |
| 压力逻辑 | 主角为什么不能不行动 | 外部威胁、时间限制、资源耗尽、关系牵连 | 主角发动机和章节推进 |
| 代价逻辑 | 每次获得要付出什么 | 受伤、失去资源、暴露身份、关系恶化 | 爽点不廉价，胜利不归零 |
| 资源循环 | 资源如何被消耗和再获得 | 钱、情报、身份、能力、人脉、装备 | 中长篇可持续 |
| 信息差逻辑 | 读者、主角、对手分别知道什么 | 误判、隐藏线索、提前透露、延迟揭示 | 悬疑、智斗、翻牌 |
| 关系牵连 | 一个行动如何改变关系网 | 盟友、敌人、组织、家人、师徒、同伴 | 人物线和后果线 |
| 规则约束 | 什么规则限制行动 | 能力限制、组织规则、副本规则、社会规则 | 解法、破绽、反制 |
| 兑现延迟 | 承诺如何分期兑现 | 小爽点、大承诺、阶段奖励、延后真相 | 节奏和追读 |
| 状态变化 | 每章/事件结束后局势是否改变 | 身份、资源、关系、敌我态势、信息状态 | 防止水文和原地打转 |
| 后果逻辑 | 赢了以后为什么不是结束 | 暴露、更强敌人、规则升级、代价回收 | 下一事件启动 |
| 反咬逻辑 | 胜利或能力如何制造新风险 | 能力副作用、敌人学习、组织注意、关系裂痕 | 阻力升级 |
| 读者追问逻辑 | 章末为什么点下一章 | 明确问题、未完成行动、即将兑现、危险临近 | 章末钩子质量 |

这些只是候选，不是定论。每一条都需要跨题材样本和反例验证。

## 4. 提取字段

每条基础逻辑候选必须记录：

```text
logic_name: 基础逻辑名称
sample_scope: 样本范围
evidence_location: 作品 / 章节 / 事件位置
observed_behavior: 原文里发生了什么
reader_function: 它让读者为什么继续看
mechanism_hypothesis: 可能的底层机制
counterexample_needed: 什么样的反例会推翻它
transfer_boundary: 哪些能迁移，哪些不能迁移
production_effect: 它会改变哪个生产动作
risk_note: 是否有相似风险或平台风险
confidence_level: 结构置信 / 样本事实置信 / clean-room 置信 / 平台反馈置信
```

## 5. 归纳门槛

一条基础逻辑不能因为“听起来对”就进入运行包。

进入候选库需要：

- 至少来自 3 本以上样本。
- 至少覆盖 2 个不同题材或 route。
- 有章节或事件级证据。
- 有至少 1 个反例或边界样本。
- 能说明它改变哪个生产动作。

进入 stable 需要：

- 能指导 clean-room 生成非贴皮文本。
- 能通过内部追读或盲审诊断。
- 不依赖某本书的高识别设定、桥段或文风。
- 能和平台约束兼容。

## 6. 当前最小闭环

当前不要直接做完整跨题材百科。

先做小闭环：

```text
选 3-5 本不同题材当前榜单样本
-> 每本只拆前 10 章
-> 标注基础逻辑候选
-> 找共性和反例
-> 回修 chapter_evidence 字段
-> 做一个 clean-room 前 3 章诊断
```

如果前 10 章字段无法稳定记录这些逻辑，说明 P2 提取协议要回修。

如果 clean-room 前 3 章无法体现这些逻辑，说明归纳还停留在口号，没有进入生产动作。

## 7. 与都市高武的关系

都市高武仍然可以是当前主样本方向。

但都市高武样本只能回答：

```text
这些基础逻辑在都市高武里如何变形？
```

不能直接回答：

```text
小说通用底层逻辑就是都市高武这套。
```

所以后续样本池应该分两条线：

- 同赛道线：都市高武 / 异常事件 / 规则破解，用于 route 深拆。
- 跨题材线：从其他当前榜单题材抽基础逻辑，用于防止系统被单赛道绑死。
