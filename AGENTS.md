# Project Agent Instructions

## 小说蒸馏执行纪律

在小说蒸馏、作者建模、路线归纳和 clean-room 复刻任务中，第一目标不是“写得顺”，而是：

`先把作者 / 单书 / route / 推进系统分清楚，再写。`

硬规则：

- 不能把小说理解成“更长的文案”。
- 不能只抽文风，必须同时抽承诺、推进、阻力、揭示和回报。
- 不能把同一作者不同书的商业 route 误判成不同作者。
- 不能把所有题材硬压成一个母模板。
- 不能让 blind review 直接变成源规则；必须回原文证据确认。
- 不能在复刻阶段偷偷用执行者自己的写作趣味“优化”目标作者。
- 不能只做书级判断，必须区分书级 / 卷级 / 章节级 / 场景级。

## 当前默认分层

小说研究默认按下面几层拆：

1. 作者层
2. 书级承诺层
3. 卷 / 单元层
4. 章节推进层
5. 场景因果层
6. 语言渲染层

## 当前默认验证问题

小说复刻或试写至少回答：

1. 像不像目标作者或目标 route
2. 愿不愿继续看下一章
3. 哪一段开始掉速
4. 翻牌是否有前铺，还是临时编造

## 仓库边界

- 本仓库优先管理方法、文档、模板、脚本和小样本。
- 大体量原始小说语料默认放仓库外部，只在文档中记录来源和路径。
- 需要新建顶层结构时，优先先写 docs 说明，再扩代码骨架。
