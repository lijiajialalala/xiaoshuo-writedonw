# Single Chapter Diagnostic Packet：资源兑现 -> 正式任务入口章

## Source Scope

```yaml
source_family: "S031 chapter 9 diagnostic only"
source_basis: "第8-10章 source-open structural reading"
packet_status: "diagnostic_only"
not_for_production: true
```

这个 packet 只用于 clean-room 单章复刻诊断，不是最终小说生产 skill。

## Target Chapter Role

写一章“上一轮胜利后的资源兑现章”，但不能停在领奖。

本章必须完成：

```text
上一轮胜利/考核/任务成果
-> 获得资源或权限
-> 把资源转成新的能力、线索或行动资格
-> 立刻接入第一个正式任务/案件/外部压力
-> 章末推出下一章的搭档、现场、对手或调查入口
```

## Chapter Entry Contract

开章必须接一个已经完成的胜利结果。

不要从空白日常开始。

需要让读者立即知道：

```text
主角刚刚证明过自己。
组织/系统/上层现在愿意给他资源。
这个资源如果兑现，会让主角进入下一层任务。
```

## Beat Count And Scene Shape

建议 4 个 beat。

### Beat 1：资源选择

```text
主角被带到资源入口。
普通资源看起来很多，但选择复杂。
一个资源掌握者或制度入口给出高价值建议。
主角判断它对自己有利，接受。
```

### Beat 2：资源消化

```text
主角立刻使用资源。
写短暂但有功能的变化画面。
不要铺成修炼百科。
必须说明变化带来的新能力、新权限或新解法。
```

### Beat 3：任务入口

```text
主角打开组织/系统/任务池。
挑中一个和新能力存在功能呼应的任务。
任务必须有奖励，也必须有最低限制或流程要求。
```

### Beat 4：章末外推

```text
主角因为限制不能立刻单人解决。
他必须招募、申请、报备、找搭档、拿许可或进入公开频道。
公开动作引发反应。
章末停在“下一章必须兑现谁来/去哪/怎么查”。
```

## Micro Goal And Conflict

主角微目标：

```text
把刚得到的资源尽快转成能用于下一次行动的东西。
```

本章冲突不是大反派，而是轻阻力：

```text
资源选择复杂。
资源必须被消化，不是拿到就强。
正式任务有流程、队伍、权限或最低条件。
```

轻阻力必须真实改变行动。如果没有它，下一章就不会被逼出来。

## Opposition Behavior

本章阻力可以来自：

```text
组织制度
任务规则
权限门槛
搭档要求
资源消化成本
公开身份带来的围观或误判
```

阻力不需要强到威胁生命，但必须让主角不能“拿完奖励直接去打怪”。

## Information Release Plan

按这个顺序放信息：

```text
1. 上一轮胜利给主角带来的资源资格。
2. 高价值资源为什么比普通资源强。
3. 资源消化后主角具体多了什么能力/权限/解法。
4. 正式任务的奖励和最低限制。
5. 任务背后有一个未解问题。
6. 章末放下一章的行动入口，不解释真相。
```

不要提前解释任务真相。

## Payoff And Delta Rules

本章必须有两个 delta。

### resource_power_delta

至少改变一项：

```text
能力
权限
道具
资源额度
任务资格
情报入口
```

### visibility_legitimacy_delta

至少改变一项：

```text
谁知道主角更强了。
谁开始误判或关注主角。
主角是否获得公开接任务的资格。
主角是否从新人变成被人申请合作的对象。
```

## Chapter Exit Hook

章末不要写总结。

可选出口：

```text
公开招募引发大量申请。
系统/组织频道有人质疑，下一章立刻反转。
某个功能型搭档出现。
任务现场传来新情报，逼主角马上出发。
```

章末要让读者问：

```text
谁会加入？
任务现场到底有什么？
新能力能不能用？
主角公开动作会带来什么反应？
```

## Minimum Language Renderer

```text
短段推进。
每个设定说明后必须接主角判断或行动。
主角内心以利益判断、行动判断、风险判断为主。
资源说明可以有爽感，但不能连续堆名词。
能力变化要有画面，但画面服务功能。
章末落动作或公共反应，不落抒情总结。
```

## Prohibited Shapes

不能出现：

```text
749
收容局/异常局这种明显换皮 749 名称
飞熊观想图
开窍丹/照璇/贡献点
黑老太太、灭门啃脸案
后勤部长送功法的同形人物关系
任务 APP + 论坛喊话的同形组合
大学实习生加入官方组织后接怪物任务的原路径
```

如果需要类似功能，请重构为完全不同内容形状。

## Editor Gate Notes

本章的编辑风险：

```text
容易变成福利说明书。
容易像游戏任务面板。
容易有原书换皮风险。
容易只升级不制造下一章压力。
```

必须做到：

```text
资源兑现快。
行动连续。
新任务有明确疑问。
下一章压力清楚。
没有高识别源书内容形状。
```

## Validation Gate

复刻稿合格前提：

```text
它是一章完整小说正文。
读者能看出开章承接上一轮胜利。
资源不是口头奖励，而是改变主角能力/权限/行动。
本章后半一定接入新任务/新压力。
章末推动下一章。
没有源书专属组织、能力、怪物、案件或桥段顺序。
没有 AI 水文式设定堆砌。
```
