# P2-alpha 复盘：三本前 10 章校准

## 0. 结论

P2-alpha 通过，但只能说明一件事：

```text
当前 P2 前 10 章提取器可以从三种不同样本里产出 route_candidate。
```

它不能说明：

```text
某条 route 已经可以直接生产。
某本书的开局可以被复用。
番茄都市高武就是我们的最终方向。
现在可以写正文。
```

当前允许进入下一步：

```text
P2-beta：10-15 本前 10 章正式快拆。
```

同时需要一个小回修：

```text
P2-beta 模板必须显式记录 resource_power_delta 和 visibility_legitimacy_delta。
```

原因是三本都证明，前 10 章真正让读者继续看的，不只是“发生了爽点”，而是每章都在改变资源、战力、身份、公开认知或合法性。

## 1. 三本是否填完核心字段

| 样本 | route_candidate | 是否填完 | 结论 |
| --- | --- | --- | --- |
| S031《大一实习，你跑去749收容怪物》 | 官方异常组织收容成长路线 | 是 | 可进入 L2，但必须补第一个案件闭环 |
| S033《时停起手，邪神也得给我跪下！》 | 有限超规格能力公开压力局 | 是 | 可进入 L2，但只能作为高概念 route 候选 |
| S034《天赋差？我一块钱买一年修为！》 | 资源燃料型系统升级路线 | 是 | 可进入 L2，但必须重点检查战力膨胀控制 |

P2 字段整体可用。

最有效字段：

```text
reader_promise
protagonist_goal
obstacle
protagonist_action
information_reveal
payoff
state_change
next_chapter_pressure
continuity_memory_signal
transfer_boundary
risk_note
```

最需要加强字段：

```text
state_change
```

原因是 `state_change` 太大。真实样本里，状态变化至少要拆成两类：

```text
resource_power_delta：资源、战力、能力、资金、权限、道具、情报是否变化。
visibility_legitimacy_delta：谁现在怎么看主角，主角能力是公开、隐藏、合法、非法，还是被误读。
```

## 2. 三条 route 的差异

### S031：官方异常组织收容成长路线

核心发动机：

```text
个人猎怪欲望
-> 官方组织接入
-> 考核评级
-> 任务 APP
-> 收容怪物
-> 能力 / 贡献点 / 权限增长
-> 更高任务
```

读者追的是：

```text
主角加入异常组织后，如何通过收容怪物不断拿奖励、提权限、接更高价值任务。
```

前 10 章最强证据：

```text
第1-3章先证明主角个人猎怪和收容能力。
第4-5章用 749 把个人能力接入制度、福利、任务和资源。
第6-8章用考核建立组织内战力标尺。
第9-10章进入第一个正式案件，并用辅调补调查功能。
```

L2 必须补读：

```text
第9-13章黑老太太案件闭环。
```

理由：

```text
前10章只证明组织成长循环启动了，还没证明案件单元真的能产生阻力、调查、翻牌和收束。
```

禁止迁移：

```text
749 组织壳
怪物收容监狱
民俗怪物形状
新人考核配置
任务 APP 细节
黑老太太案件
```

### S033：有限超规格能力公开压力局

核心发动机：

```text
强但受限的能力
-> 公开高压规则场
-> 主角用表演和误判拖时间
-> 能力在具体机械问题中兑现
-> 外界错误解释
-> 身份 / 资源 / 民心 / 组织压力变化
-> 下一局更大压力
```

读者追的是：

```text
主角如何用有限外挂在众目睽睽的必死局里翻盘，并把别人误判变成自己的资源。
```

前 10 章最强证据：

```text
第1-4章完成死亡赌局的公平兑现。
第5-6章把胜利转成身份、禁忌物、资源回流和背叛信息差。
第7章保留李右变量，扩展王蛇威胁。
第8-10章把异化兽战斗变成刷资源、刷民心和夺权正当性的场。
```

L2 必须检查：

```text
灰土镇第一卷事件如何闭合。
主角是否持续有新限制，而不是靠口嗨和时停碾压。
```

禁止迁移：

```text
时停能力
一分钟/累积时间设定
禁忌物名称和规则
六弹赌局
鬣狗围城
主角高识别口吻和热血台词路径
```

### S034：资源燃料型系统升级路线

核心发动机：

```text
弱者未来被交易
-> 系统把钱变成修为
-> 钱成为燃料
-> 债务和敌人变成资金来源
-> 暴涨必须合法展示
-> 暴涨必须隐藏
-> 更高级资源价格继续反咬
```

读者追的是：

```text
主角如何把钱、债、敌人和考试路径全部变成升级燃料，同时避免实力暴涨被官方和大势力发现。
```

前 10 章最强证据：

```text
第1章把成绩交易金额和系统燃料绑定。
第2-4章建立合法化展示和拒绝捷径。
第5-6章完成第一次大额充值和战力暴涨。
第7-9章让敌人根据旧情报反咬，并把敌人资金转成资源。
第10章及时把问题转向隐藏实力和高级功法价格。
```

L2 必须检查：

```text
高考闭环。
敛息诀是否控住暴露风险。
赵家反咬是否升级。
战力膨胀是否还能保持张力。
```

禁止迁移：

```text
一块钱一年修为
买成绩开局
黑贷升级
赵泽派杀手
老默送钱
敛息诀试学路径
```

## 3. 三本共同的底层机制候选

这些不是最终规律，只是 P2-alpha 候选。进入 P2-beta 后要用更多样本和反例验证。

### 3.1 读者承诺必须能转成可见行动

三本都不是只给标签。

```text
S031：749 / 收容 -> 主角必须去找怪、收容、接任务。
S033：时停 / 邪神 -> 主角必须进入公开死亡局和战场。
S034：买修为 / 系统 -> 主角必须拒绝交易、充值、隐藏、参加高考。
```

可迁移机制：

```text
主承诺必须在前 10 章内变成主角无法回避的行动链。
```

### 3.2 能力不是奖励，而是新问题制造器

三本能力兑现后都没有停。

```text
S031：收容奖励让主角进入 749 任务和考核系统。
S033：时停翻盘后带来身份、禁忌物、背叛和民心问题。
S034：系统升级后带来资金、暴露、隐藏和合法身份问题。
```

可迁移机制：

```text
每次能力兑现都必须改变资源循环，并制造下一层风险。
```

### 3.3 阻力必须根据旧状态行动

三本的反派或组织都在用旧认知判断主角。

```text
S031：749 认为主角是开窍双天赋，不知道真实金手指。
S033：谭杰以为主角使用禁忌物会被耗死。
S034：老默根据 0.8 气血旧情报入室动手。
```

可迁移机制：

```text
敌人不是站着等主角装逼，而是基于旧信息做合理行动，然后被主角的新状态反杀或误导。
```

### 3.4 前 10 章必须形成小闭环，同时打开大问题

三本都在前 10 章做了小闭环。

```text
S031：个人猎怪 -> 加入 749 -> 考核完美 -> 第一个任务启动。
S033：死亡赌局 -> 禁忌物收容 -> 异化兽战斗 -> 民心钩子启动。
S034：成绩交易 -> 系统验证 -> 杀手反杀 -> 隐藏实力需求启动。
```

可迁移机制：

```text
前 10 章不是完整第一卷，但必须让读者看到第一轮承诺已经兑现，并相信下一轮更大。
```

### 3.5 长篇一致性从第 1 章就开始

三本前 10 章都已经制造大量必须外部记录的变量。

```text
S031：怪物名册、能力奖励、749规则、贡献点、任务权限、人物认知。
S033：时停秒数、禁忌物状态、气血回流、派系认知、民心变化。
S034：余额、气血、技能、债务、赵泽知情状态、敛息诀进度。
```

可迁移机制：

```text
长篇一致性不是后期才管。前 10 章开始就要记录状态变量，否则 clean-room 前 10 章一定会乱。
```

## 4. P2-alpha 没有证明什么

### 4.1 没有证明都市高武就是最终方向

三本只证明：

```text
番茄都市高武/异常事件里，至少存在三种可提取的前10章发动机。
```

还没有证明：

```text
这些发动机比其他题材更适合我们。
这些发动机适合七猫或起点。
这些发动机能支撑我们自己的 house style。
```

### 4.2 没有证明可以开始写正文

现在缺：

```text
P2-beta 横向样本
P3 第一卷/事件链深拆
内容内核卡
clean-room 前3章诊断
相似风险盲审
最小 Story Bible / Style Bible
```

### 4.3 没有证明可以复用任意桥段

三本的强点恰恰也是相似风险最高的地方。

```text
S031 的 749 / 收容 / 民俗怪物不能搬。
S033 的时停 / 赌局 / 禁忌物 / 鬣狗围城不能搬。
S034 的一块钱修为 / 买成绩 / 黑贷 / 敛息诀不能搬。
```

我们能抽的是：

```text
组织如何提供任务和资源。
公开压力局如何制造误判。
资源燃料如何驱动升级和反咬。
能力兑现后如何制造下一层问题。
```

## 5. critical-thinking-mode 自问自答

### Q1：为什么说 P2-alpha 通过？

因为三本都填完了核心字段，并且每本都产出了可解释的 route_candidate。

更重要的是，三本不是只拆出设定，而是拆出了：

```text
主角为什么必须行动
阻力如何反咬
能力如何兑现
兑现后状态如何变化
下一章为什么还要看
哪些内容不能迁移
```

如果只拆出“749、时停、买修为”这些设定，就不能算通过。

### Q2：为什么不直接选其中一条 route 开写？

因为 P2-alpha 只有三本，样本数量不足。

三本的作用是校准提取器，不是决定最终项目。

现在直接开写，会有两个风险：

```text
把单本高识别桥段误当成可迁移机制。
把番茄当前榜单偏好误当成小说通用能力。
```

所以只能进入 P2-beta 或指定 L2 深拆，不能直接生产。

### Q3：为什么不继续扩大样本之前先回修字段？

因为三本都暴露了同一个问题：

```text
state_change 太粗。
```

真正有效的状态变化至少有两类：

```text
资源/战力/权限是否变化。
公开认知/合法性/秘密状态是否变化。
```

如果 P2-beta 不显式记录这两类变化，后面会把“剧情推进”写成泛泛总结。

### Q4：有没有可能我们只是偏爱强设定爽文？

有。

三本都是番茄都市高武榜单样本，天然偏向强设定、快爽、明确系统和高频反转。

修复措施：

```text
P2-beta 必须加入 counter_preference_sample。
P2-beta 必须加入 cross_route_control_sample。
顶级作者/IP样本只做 top_benchmark_observe，不占普通样本结论。
```

### Q5：有没有可能三本共同点只是番茄平台偏好，不是小说底层逻辑？

有。

例如：

```text
前三章强钩子
十章内强兑现
敌人频繁送脸
强数值反馈
```

这些可能是番茄当前平台机制强化出来的。

修复措施：

```text
P2-beta 后需要用七猫/起点或跨题材样本做反向对照。
但当前阶段先不扩平台，因为 P2-alpha 的目标只是校准番茄专项提取器。
```

### Q6：为什么不把“主角必须行动”单独当成一个大模块？

可以，但现在不能提前定名为某一个解释。

三本里它至少有三种形态：

```text
S031：主角必须行动，因为怪物收容是他变强和进入组织的唯一通道。
S033：主角必须行动，因为公开压力和生存危机会立刻杀死或吞掉他。
S034：主角必须行动，因为高考倒计时、赵泽压迫和系统秘密共同逼迫他合法化暴涨。
```

所以更准确的候选名是：

```text
持续行动发动机。
```

它不是单独等于“不得不变强”，而是包含生存、资源、身份、合法性、关系和承诺兑现。

### Q7：当前最危险的误判是什么？

最危险的误判是：

```text
把 route 当题材标签。
```

错误写法：

```text
我们要写 749。
我们要写时停。
我们要写买修为。
```

正确写法：

```text
我们要研究官方组织如何把异常事件转成任务循环。
我们要研究有限能力如何在公开压力局里制造误判和翻盘。
我们要研究资源如何成为系统升级燃料，并被债务、敌人和隐藏需求反咬。
```

### Q8：现在能进入运行包的是什么？

现在只能进入候选库，不能进入稳定运行包。

候选机制包括：

```text
承诺必须转行动。
能力兑现必须制造新问题。
敌人根据旧状态行动。
每章必须产生资源/战力/认知/合法性变化。
前10章必须完成小闭环并打开大问题。
长期一致性变量必须从第1章开始记录。
```

进入稳定运行包之前，还要经过：

```text
P2-beta 横向验证
L2 事件链深读
clean-room 前3章/前10章诊断
相似风险盲审
```

## 6. P2 协议回修项

P2-beta 前新增两个字段。

### resource_power_delta

记录这一章结束后，主角或故事系统是否获得、失去或改变了：

```text
钱
战力
能力
权限
身份
道具
情报
任务入口
组织资源
关系资源
```

判断标准：

```text
如果这一章删掉，后续资源/战力/身份不受影响，那它的 resource_power_delta 很弱。
```

### visibility_legitimacy_delta

记录这一章结束后，主角的公开状态是否变化：

```text
谁知道了主角的新能力。
谁误判了主角。
主角能力是公开、隐藏、合法、非法，还是被包装成别的原因。
主角在组织、公众、敌人、同伴眼里的正当性是否变化。
```

判断标准：

```text
如果这一章只让主角变强，但没有改变任何人的认知和后续行动，它对长篇压力的贡献不足。
```

## 7. 是否允许扩到 P2-beta

允许，但带条件。

P2-beta 必须做到：

```text
10-15 本，不只选我们喜欢的书。
至少包含 1-2 本 counter_preference_sample。
至少包含 1-2 本 cross_route_control_sample。
继续保留 top_benchmark_observe，但不让强作者/IP样本污染普通结论。
每本必须写 transfer_boundary 和 risk_note。
每本必须记录 resource_power_delta 和 visibility_legitimacy_delta。
```

不允许做：

```text
直接写正文。
直接做 runtime skill。
直接把三条 route 合并成一个“都市高武模板”。
直接把 S031/S033/S034 的高识别内容形状迁移进我们项目。
```

## 8. 下一步

下一步按这个顺序：

```text
1. 回修 schema 和 P2 协议，加入 resource_power_delta / visibility_legitimacy_delta。

```

如果中途发现 P2-beta 字段仍然抓不住“为什么读者继续看”，停止扩样本，先回修提取器。
